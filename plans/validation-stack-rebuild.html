How will agents change the stack without overlapping ownership?

waves

Wave A · inventory

Read-only agents produce the old-commit allocation ledger and dependency boundary audit. No implementation begins until every old commit is assigned, split, or dropped.

Wave B · foundations

PRs 1–3 may use separate agents only when their working directories and file ownership are disjoint. Each works from the freshly validated parent.

Wave C · canonical path

PRs 4–8 run sequentially because optimization, environments, adapters, and runner integration overlap. One agent owns each branch and commit.

The requested Opus/Sonnet tier labels are unavailable in the Codex agent interface; default frontier agents handle architecture and implementation, followed by an independent reviewer.

What must pass before a branch can become a PR?

gates

required Static checks

  • uv run ruff format --check .
  • uv run ruff check
  • uv run ty check
  • The installed pre-commit and pre-push hooks invoke the same script.

gate Tests and time

  • Focused tests prove the PR’s public contract.
  • The recorded wall time is strictly below 60 seconds.
  • PR 8 also runs fake-transport cells for every optimizer.
  • Slow tests remain a separately declared validation obligation.

review Architecture

  • Parent-to-branch diff contains only the declared responsibility.
  • Lower layers do not import runner modules.
  • Only canonical adapters make optimization decisions.
  • No old implementation name remains in code, tests, or docs.

review PR record

  • Title states the delivered behavior.
  • Description explains why, ownership, evidence, and stack position.
  • Validation todo identifies any check the owner must still witness.
  • Removed tests appear in a named ledger with replacement coverage.
