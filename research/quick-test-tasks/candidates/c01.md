# c01 · Invented-Glyph Positional Base Arithmetic

**Status: GATED**

You know this as: the counterfactual non-decimal base-arithmetic probe from *Reasoning or Reciting?* — dressed in a per-instance invented glyph alphabet so the model cannot fall back on memorized base tables.

---

## Verdict

**Gated in round 2 — refuted by both round-1 skeptic lenses at major severity, with no concrete rescue.** The task is structurally attractive: a ten-line positional evaluator gives exact ground truth, per-instance random legends make it contamination-proof, and the literature documents a genuine default-prompt floor. But it is trapped between two failure modes that its own design cannot escape at once. When the base and legend are stated explicitly, a single competent instruction ("read the glyph→digit map, apply place weights") one-shots the pool, collapsing the incremental gap that criterion 4 requires. When the quirks are left implicit, there is no writable reference prompt that reaches roughly full accuracy, so there is no known ceiling to optimize toward. Compounding this, the natural inverse (encode a decimal into the glyph system) has non-unique ground truth once null-glyph and signed-digit strata are active, which breaks exact-match scoring. The gate is a round-1 verdict-health decision, so it applies regardless of the implementation score: the only fixes are structural redesigns that turn this into a different candidate (closer to c03's opaque keyed operators). The implementation score is recorded for completeness but does not lift the gate.

## What it proposes

### Evaluate a numeral written in a per-instance invented glyph alphabet under a nonstandard positional base

A seeded generator emits one or two numerals in an **invented glyph alphabet** together with an inline **legend** (glyph→digit map) and a **base spec** that varies per stratum: base 9 or 11 (bases 8 and 16 are avoided as pretraining-frequent), mixed-radix, balanced or signed digits, MSB- vs LSB-first ordering, and a null-glyph position skip. The model returns a **single decimal integer** (or a fixed-width glyph string), scored by exact match against a roughly ten-line positional evaluator. The latent rules — legend parse, place weights, signed-digit handling, endianness, null glyph, carry variant — are stratified across the pool so that no fixed one-shot instruction covers every quirk at once, and per-instance random legends make the pool contamination-proof.

### Example instance (illustrative)

Constructed faithfully from the merged summary; not a run-verified generator output. Seeded conditions: base 9, glyph legend `{◇=0, ▲=1, ○=2, ✦=3, ◈=4, ▽=5, ◐=6, ✕=7, ◭=8}`, MSB-first ordering.

```
Legend: ◇=0 ▲=1 ○=2 ✦=3 ◈=4 ▽=5 ◐=6 ✕=7 ◭=8
Base: 9, most-significant glyph first.
Numeral: ○▽✕
Q: Give the value as a single decimal integer.

Gold: 214   (2·81 + 5·9 + 7 = 162 + 45 + 7 = 214)
Naive-prompt miss: a base-10 reading of "257" → exact-match 0;
a "convert to base-10 then back" attempt that ignores the invented
legend also misses.
```

## Perspectives

### Round-1 rubric fit
- **Supports:** Scored 27/28; only criterion 4 (not one-shot guessable) is soft. Exact single-integer scoring, contamination-proof legends, clean latent-rule decomposition, diagnostic per-rule failures.
- **Against:** The soft criterion is the decisive one — positional evaluation is a known skill, so a competent prompt approaches the ceiling. Landscape flags this as the shared weakness of the invented-notation cluster.

### Optimization-path skeptic
- **Supports:** If per-stratum conventions are kept secret and demo coverage is capped, individual quirks (endianness, null glyph, signed digits) form a real multi-rung ladder in principle.
- **Against:** Refuted at major severity — with explicit specs a single instruction one-shots the pool; the gap the optimizer is supposed to close does not survive an honest ceiling prompt.

### Measurement skeptic
- **Supports:** Ground truth from a tiny reference evaluator is deterministic and cheap to verify.
- **Against:** Refuted at major severity — with implicit quirks there is no writable roughly-full-accuracy reference prompt, so there is no known ceiling; and the encode direction has non-unique ground truth, breaking exact match.

### Implementation lens (round 2)
- **Supports:** The reasoning-gym infrastructure is clean and run-verified (seeded, deterministic, re-seedable); the arithmetic oracle is trivial. I1 held at 2 on a confirmed run.
- **Against:** I6 is 0 — the refutation debt is not a bounded repair but a structural redesign, which is exactly what gates the candidate. The invented-glyph legend is net-new code, not in-repo.

### Research currency
- **Supports:** Counterfactual/non-standard arithmetic and in-context symbol definition are actively studied through 2025-2026 (In-Context Algebra, GSM-Symbolic re-evaluations, length-generalization); TB 2.
- **Against:** Yuan et al. overlaps the anchor rather than extending it, and the balanced-ternary and length-generalization sources are training-side, not prompt-optimization evidence — the "prompt closes the gap incrementally" claim is untested here.

## Implementation: steps & risks

1. **Subclass the reasoning-gym generator.** `base_conversion` ships standard-glyph base conversion with no invented legend; write a new `ProceduralDataset` subclass (~80-120 LOC) that samples value + base with `rng = Random(seed + idx)`, samples an invented-glyph permutation from a nonce alphabet, and renders the source numeral under the legend.
2. **Build the oracle and strict scorer.** Compute the gold via standard integer conversion mapped through the same legend; override `score_answer` for strict 0/1 exact match (~10 LOC), since the base class gives substring partial credit.
3. **Stratify the latent rules.** Expose base value, endianness, null-glyph skip, and signed/balanced-digit family as config fields so no single instruction sentence covers the pool; avoid bases 8/16; cap demo coverage so few-shot cannot leak the whole convention set.
4. **(Never reached) rescue gates.** A pass would require a red-team check that no fixed one-shot prompt matches the ceiling and a resolution of the encode-direction non-uniqueness — but these are structural redesigns, so the candidate is gated before this step.

### Risks

- **One-shot collapse:** an explicit-spec prompt reaches the ceiling in a single call, erasing the incremental gap (the major optimization-path refutation).
- **No known ceiling under implicit quirks:** without a writable roughly-full-accuracy reference prompt, the measurement story has no anchor (the major measurement refutation).
- **Non-unique encode ground truth:** the decimal→glyph direction admits multiple correct strings once null/signed strata are on, so it cannot be scored by exact match.
- **Decimal round-trip shortcut:** CoT-capable models can "convert to base-10 and back"; invented glyphs and bounded output are the only mitigations, and they only partly work.
- **Shared-logic oracle:** the reasoning-gym oracle reuses the generator's own conversion routine, so a shared bug would corrupt prompt and answer symmetrically (no independent cross-check).

### Repo review & run verification

- **Codebase:** [open-thought/reasoning-gym](https://github.com/open-thought/reasoning-gym) (Apache-2.0) — deep-read notes at [../repos/open-thought-reasoning-gym.md](../repos/open-thought-reasoning-gym.md). Seed plumbing is threaded and clean (`Random(self.seed + idx)` per item); the invented-glyph semantics are net-new (~250-350 LOC in our tree, 0 lines changed in reasoning-gym).
- **Run verification: verified (partial adaptation).** On 2026-07-21 the `base_conversion` generator ran, was byte-identical at seed 42 across runs, varied at seed 123, and passed ground-truth spot checks — confirming the infrastructure. The invented-glyph task layer itself was never run-verified because it is not yet written; I1 was held at 2 for exactly this reason.
