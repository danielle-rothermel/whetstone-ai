What does the task ask the model to do?

What it proposes

A grammar-violation classifier where the MC letter names the broken rule

A seeded generator samples strings from a randomized formal grammar composing four to five constraint types — feature-agreement pairs, nesting-depth limits, forbidden adjacencies, balanced multi-type delimiters, and one attribute-dependent depth limit — and either keeps a string well-formed or injects exactly one violation of a known class. The model outputs a single multiple-choice letter: A well-formed / B agreement / C nesting / D adjacency, with an earliest-violation-by-scan-order tie-break when a corruption could read as more than one class. Ground truth is the injected class; the score is exact letter match. Because each constraint the prompt captures catches exactly one violation class, the strata are cleanly incremental at the pool level, and the label directly names the violated rule, making failures maximally diagnostic. Randomized grammars per run make instances contamination-proof, and the four-way format gives a one-in-four guess floor.

Example instance

Illustrative, built faithfully from the merged summary. Seeded conditions: this run draws an agreement rule (paired tags must share their height feature), a max nesting depth of two, and a forbidden adjacency ('q' may not immediately follow 'p'). One violation is injected; the tie-break is earliest-by-scan-order.

Grammar (this run):
  R1 agreement : paired tags must share their height feature (+hi with +hi)
  R2 nesting   : bracket depth may not exceed 2
  R3 adjacency : symbol 'q' may not immediately follow 'p'

String:  [ a+hi  [ b-hi  a+hi ] ]  p q

Question: which rule class does this string violate?
  A well-formed   B agreement   C nesting   D adjacency

Gold: D  (the 'p q' adjacency breaks R3; depth stays within 2, tags agree)

Naive-prompt miss: a one-shot solver that only internalized "check brackets
balance and nest" scans depth, sees depth 2 is fine, and answers A well-formed
— it never learned the per-run adjacency and agreement rules, so it catches at
most two of the four classes.

Where does this task come from?

Academic anchor & lineage

Literature parents

  • MLRegTest (van der Poel et al., JMLR 2024) — a benchmark of 1,800 regular languages across the subregular hierarchy scored as accept/reject membership; the core source for c06's rule-strata design, supplying the subregular families that become the violation classes.
  • Self-Attention Networks Can Process Bounded Hierarchical Languages (Yao et al., ACL 2021) — the foundational Dyck-n / bounded-depth bracket expressivity result that anchors the next-legal-bracket and nesting variants; it is the concrete anchor for the "Dyck membership & next-legal-bracket" research line named in the merge.
  • Training Neural Networks as Recognizers of Formal Languages / FLaRe (Butoi et al., ICLR 2025) — releases the FLaRe benchmark across the Chomsky hierarchy; the second reusable donor. Correction: the corpus dates it "2024" and implies van der Poel authorship, but it is ICLR 2025, first author Butoi — a distinct paper from MLRegTest.

Brainstorm parents

  • Artificial Grammar Well-Formedness Judgment — the seed brainstorm item; became the well-formed / violated framing.
  • Symbolic Sequence Categorization (SSC) — supplied the multi-class categorization label shape; Nested-Bracket Path Resolver contributed the depth/bracket variants and the next-legal-bracket alternate task.
  • Merged entry c06 — origin: both brainstorm and literature; unifies the well-formedness judgment with subregular-membership benchmarks.

Dead branches: dropped near-relatives

  • D3 TokenCounter (distinct vowel/consonant counting) — letter counting is tokenization-bound and StringLLM/CUTE show char-count tasks are noisy at temp 0, threatening exact-match; a live constraint on why c06's violations must be structural, not count-based.
  • D9 Char-level string-task cluster (StringLLM, CUTE, letter-counting) — not a task, kept only as the red-flag design caveat that justifies MC / closed-vocab framing over character-level scoring in this family.
  • Pure Dyck-k balanced-membership — retained only as a variant: balanced-parenthesis membership is one skill (counting depth), so rule decomposition is thin unless enriched with multiple bracket types and forbidden-substring constraints — the enrichment c06 adopts.
  • Convergent cousins c21 / c03 — not dropped, but named in the round-2 reason as doing the hidden-rule-table job more cleanly; c06's rescue moves headroom onto hidden tables and lands on their territory.

What live 2025-2026 research backs it?

  • Diagnosing CFG Interpretation in LLMs / RoboGrid (Li et al., arXiv 2026) — directly tests LLMs on novel CFGs at inference and stress-tests recursion depth and complexity, which is exactly the c06 nesting / grammar-from-context threat model.
  • Causally Evaluating the Learnability of Formal Language Tasks (Snaebjarnarson et al., arXiv 2026) — builds formal-language tasks from probabilistic finite automata and argues correlational membership evals are flawed, speaking to c06's honest-ladder and measurement concern.
  • Creating ConLangs to Probe Metalinguistic Grammatical Knowledge of LLMs (Taguchi & Sproat, arXiv 2025) — synthetic-grammar well-formedness probing of LLMs, adjacent to c06's agreement/violation judgment.

What does it take to build, and what can go wrong?

Implementation: steps & risks

Vendor automata + oracle, not samplers

Take the ~1,470 committed .fst files and the three-line composition oracle (pynini.acceptor(s) @ fsa non-empty == accepted) from heinz-jeffrey/subregular-learning; the 13-family filename tag becomes the latent-rule strata. Repo edits: zero — treat it as a data + oracle donor

Write a seeded sampler

Author c06_gen.py (~90 LOC, outside the repo) with a threaded random.Random(seed) drawing fixed-length strings over each FST's alphabet, because neither donor's sampler is controllably re-seedable

Build the missing violation taxonomy

For each of at least four named rules, author a rule-specific corruption operator that breaks exactly that rule while satisfying the others (this exists in neither repo — negatives are undifferentiated), and label every string via the independent oracle, not the tautological training labels

Balance and stratify

Balance labels across the four classes; compose four to five constraint types including one attribute-dependent depth limit so a one-shot prompt catches only two or three of four classes; wire the earliest-violation-by-scan-order tie-break

Red-team the floor

Gate on non-mnemonic instantiations and recompute labels, then adversarially probe for a convention-exploiting prompt that lifts the floor toward sixty percent; admit only strata that clear the gate

Risks

  • Mnemonic rule instantiations let a convention-exploiting prompt floor the task around half to sixty percent, collapsing the headroom unless the red-team gate is enforced per stratum.
  • The rescued design shifts difficulty onto hidden rule tables and converges on c21/c03, so the marginal value over already-adopted candidates is thin.
  • Heavy native dependency: the MLRegTest oracle needs pynini==2.1.2 (conda-only OpenFst binding), the main operational risk, unless the automata are exported to a pure-Python FSA format.
  • The FLaRe donor requires a two-stage offline pipeline (save_automaton.py then prepare_sampler.py) plus an SSH-URL rau git dependency before any string is produced.
