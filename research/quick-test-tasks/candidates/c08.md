# c08 · Rosetta-to-Match-Up MC Linguistic Puzzles

**Status:** RANKED #15 of 20 (round-2 rescore)

You know this as: **linguistic-puzzle rule induction (multiple choice)** — a seeded conlang grammar rendered as glossed example pairs plus a 4-way multiple-choice query, where every distractor differs from the truth by exactly one latent rule.

## Verdict

**Officially ranked #15 of 20, a slight upward drift from its round-1 26-tier standing.** c08 carries the cleanest scoring story in the whole rule-induction cluster: converting Rosetta-style translation into Match-Up multiple choice collapses noisy free-text morphology into a single-letter exact match, and the distractor construction gives a built-in incremental ladder where each captured rule eliminates one wrong answer. The design is admired but not adopted because the implementation floor is high — the only reusable codebase, LINGOLY, is a frozen scoring harness over hand-authored UKLO puzzles with no generator, no sampling, no seed, and no distractor logic, so the load-bearing engineering (a re-seedable conlang generator with latent-rule strata and a rule-based oracle) must be written essentially from scratch. That reuse penalty, priced through a run-verification-capped I1, is why a design this rubric-clean settles in the mid-lower pack; it edges just below sibling c09, which salvages more code.

## What it proposes

### Match-Up MC that turns translation difficulty into single-letter exact match

A generator seeds a fresh conlang grammar per run — nonce roots plus affixes marking word order, agreement, affix meaning, and a diacritic/tone layer with a marker-precedence hierarchy (tone resolves a homograph, a context particle overrides tone). Each instance presents **N glossed example pairs** plus one held-out query as a **4-way multiple choice** asking which pairing or meaning is correct. The **distractors** are constructed so each differs from the truth by exactly **one latent rule**, so every rule the model actually induces eliminates one distractor — a built-in incremental ladder with diagnostic wrong answers. Output is a single letter, scored by exact match, and the whole pool is contamination-proof because the grammar is regenerated each run.

### Example instance (illustrative)

*Seed note: constructed from the merged summary; a two-rule instance where the SOV word-order rule and the case-suffix agreement rule each isolate one distractor.*

```
Glossed pairs (conlang → English):
  tam-uk  belu-na   →  the child sees the dog
  belu-uk tam-na    →  the dog sees the child
  mira-uk tam-na    →  the bird sees the child

Query: which English meaning matches "tam-uk mira-na"?
  A. the child sees the bird      <- gold
  B. the bird sees the child      (violates: -uk marks subject, not position)
  C. sees the child the bird      (violates: English SVO output order)
  D. the child sees a bird        (violates: article/definiteness rule)

Gold: A
Naive-prompt miss: a model anchoring on surface word order picks B,
reading the first noun as subject and ignoring the -uk/-na case suffixes.
```

### Dead branches: dropped near-relatives

- **D1 — Inverted Multiple-Choice convention** — dropped because both sources flag HIGH one-shot risk: a single sentence ("answer the alphabetically-first incorrect option") closes the gap, failing the anti-one-shot criterion. c08 keeps the MC *format* but earns difficulty from latent-rule distractors rather than a label-inversion trick.
- **D9 / D3 — char-level string-task cluster (StringLLM, CUTE, letter-counting)** — dropped as "not a task"; kept only as the red-flag caveat that char-count and free-text-morphology scoring is noisy at temp 0. This caveat is exactly why c08 routes to single-letter MC rather than free-text translation.
- **c07 Case-Marked Micro-Conlang Translation** (refuted cousin, now GATED) — the free-text conlang translation sibling was refuted on the optimization path: its gloss table hands the latent morphology to the model in-context and its "hard rule" (case beats word order) is the highest-prior rule in linguistics; the rescue is a structural MTOB-style redesign. c08's MC framing and one-rule-per-distractor construction are the response to that refutation.

## Perspectives

### Round-1 rubric fit
- **Supports:** Distractors each differ by exactly one latent rule, giving a built-in incremental ladder with diagnostic wrong answers; MC framing tames scoring into clean 0/1 exact match. Rated a strong research-current dark horse in the round-1 landscape.
- **Against:** The exact-match arithmetic/simulation tier simply outscored it at 26/2; a clean design was not enough to reach the 28-tier finalist band.

### Optimization-path skeptic
- **Supports:** Because each distractor isolates one rule, an optimizer that surfaces one more rule (word order, agreement, affix meaning, diacritic override) measurably eliminates one distractor — an unusually legible gradient for prompt optimization.
- **Against:** No dedicated optimization-path skeptic pass was run below the 28-tier finalists, so this lens was never adversarially examined for c08; the inference above is from the design, not a skeptic verdict.

### Measurement skeptic
- **Supports:** The Rosetta-to-Match-Up conversion is a deliberate answer to the char-level / free-text noise caveat — single-letter output sidesteps the temp-0 morphology-scoring noise that sinks free-text translation.
- **Against:** No measurement skeptic pass was run for c08 below the finalist tier; distractor quality (that each really does isolate exactly one rule, with no unintended second cue) is asserted by design and flagged for pilot validation, not yet verified.

### Implementation lens (round 2)
- **Supports:** Oracle independence is strong (I2=2) — a rule-based solver over a seeded grammar is trivially hand-verifiable; redesign debt is low (I6=2) since no structural skeptic refutation is outstanding.
- **Against:** The reuse floor is the binding weakness: LINGOLY offers no generator, no seed, no distractor logic, and its dataset is CC-BY-NC-ND (no-derivatives), so I1 caps at 1 and ~300-600+ LOC must be written from scratch borrowing under 30 LOC.

### Research currency
- **Supports:** The LingOly line is unusually active into 2026 — LingBench++, LingOly-TOO, Rosetta-to-Match-Up, ConlangCrafter — so a synthetic generator here has a natural path to becoming its own research artifact.
- **Against:** Every live artifact in the line is either a frozen human-authored set or a translation/generation study; none ships the re-seedable Match-Up generator this candidate needs, so currency does not lower the build cost.

## Implementation: steps & risks

1. **Build the seeded conlang engine.** Sample a fresh grammar per run — nonce lexicon plus strata for word order, agreement, affix meaning, and the diacritic/tone precedence hierarchy — with seed plumbing throughout (absent entirely in LINGOLY).
2. **Render Rosetta pairs and the Match-Up MC builder.** Produce N glossed example pairs plus a held-out query, then construct three distractors where each perturbs exactly one latent rule and one option is the gold pairing.
3. **Write the rule-based oracle and single-letter scorer.** Derive the gold answer from the latent grammar (not a pre-stored string), reusing only LINGOLY's `clean_answer` normalization (~20 LOC) and the `safe_exact` idea (~5 LOC).
4. **Pilot-validate distractor quality.** On a batch, confirm each distractor isolates exactly one rule and no option carries an unintended second cue — the load-bearing engineering per the merged adaptation note.

### Risks
- **Rebuild-from-scratch cost.** The load-bearing generator, distractor logic, oracle, and seed plumbing do not exist in the repo; estimated ~300-600+ LOC written outside it, borrowing under 30 LOC. This is the fact that caps I1 at 1.
- **Distractor leakage.** If a distractor accidentally violates two rules (or none cleanly), the "one rule eliminates one distractor" ladder degrades and the diagnostic signal is lost — hence the mandated pilot gate.
- **License constraint.** LINGOLY's puzzle dataset is CC-BY-NC-ND-4.0 (no-derivatives), so even seeding strata from the human puzzles is legally constrained; the generator must be fully synthetic.

### Repo review & run verification
- **Codebase:** [am-bean/lingOly](https://github.com/am-bean/lingOly) (dataset CC-BY-NC-ND-4.0; code CC-BY-SA-4.0) — deep-read notes at [../repos/lingoly-am-bean-lingoly.md](../repos/lingoly-am-bean-lingoly.md).
- **Run verification:** never run-verified. The I1=1 provisional was kept as read (not run-tested); there is no generator active path to verify — the repo is a frozen-dataset scoring harness with no randomness, no seed, and a tautological pre-stored oracle.
