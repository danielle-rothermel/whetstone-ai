## What it proposes

### Task: extract one house-normalized field from a record in an invented markup

The generator emits synthetic records written in invented markups — **sigil-tagged fields**, fixed-width columns with implied decimals, custom-escaped delimiters, or one of several **dialects** signaled by a header token — and each instance asks for exactly ONE field, returned in **normalized house form** (integer-cents to decimal, dotted enum code via most-specific-wins). The output is a single short token, scored 0/1 by exact match against the generator's own record, with strictly one field per instance so no intra-example partial credit leaks in. The latent rules — the sigil-to-field map, per-dialect parsing conventions, column widths, escape handling, per-field normalization, and enum precedence — are each stratified across the pool, so pool accuracy is meant to climb as a prompt captures more rules. Sigils, widths, delimiters, and enum vocabularies are randomized per run, which makes the pool contamination-proof and resistant to one-shot guessing.

### Example instance

*Seeded conditions (from the round-1 corpus): dialect `VEL`; `ᚠ` marks unit price with the implied decimal at 3 places this run; `ᚷ` marks a date; a second money-shaped field `ᚢ` is a same-type distractor.*

```
Record: ⟦hdr:VEL⟧ ᚠ7203 ᚢ1150 ᚷ0925
Query:  unit price, in house form.

Gold:   7.203
        — the sigil map picks ᚠ over the same-type distractor ᚢ;
          this run's implied-decimal position is 3.

Naive extraction answers 7203 or 72.03  →  0.
```

## Perspectives

### Round-1 rubric fit
- **Supports:** earned a top rubric score (28, TB 2) and stood among the seven verified finalists; per-run randomized sigils/widths/vocabularies are contamination-proof and genuinely resist one-shot prompts, and strictly one field per instance keeps scoring cleanly binary.
- **Against:** the top score masked a measurement story that the adversarial lens refuted as originally specified — the raw spec included strata (checksum, character-offset widths, fixed-decimal normalization) that failed the criteria and had to be surgically removed.

### Optimization-path skeptic
- **Supports:** the survived verdict was "not refuted at major severity" — the spec-dump reference prompt defines a reachable ceiling and the latent rules genuinely stratify, so a gap-recovery ladder exists in principle.
- **Against:** a semantic type-matching prompt ("find the field matching the queried type") solves the sigil stratum for free unless every record carries at least two same-type distractors; and COPRO's score-only proposer cannot reach per-run randomized rules, so the optimizer axis must be scoped to plumbing validation.

### Measurement skeptic
- **Supports:** once the offending strata are removed, the remaining single-token exact-match design is deterministic and hand-verifiable, and the illustrative instance shows a clean naive-miss versus gold separation.
- **Against:** refuted-as-specified at major severity — cents-to-decimal at a fixed position is world knowledge rather than latent headroom, and per-instance single-token failures are the least individually diagnostic in the set: diagnosis has to be read at the batch level, not the example.

### Implementation lens (round 2)
- **Supports:** the target task shape is simple (one sampler, one short-token oracle) and the exact-match scorer is trivial once the record format is fixed.
- **Against:** no reusable codebase exists — every discovered repo was a frozen dataset, LLM-driven, or the wrong shape (I1=0, double-weighted), so the whole generator is a from-scratch build; and it carried the most stratum surgery of the finalists (I3=0), which is why its total lands at the bottom (impl 4/16).

### Research currency
- **Supports:** structured extraction and format-constrained output are live 2025 topics — StructEval and JSONSchemaBench both show real headroom even for strong models, so the underlying phenomenon is current and publishable.
- **Against:** the currency lives mostly in sibling candidates (JSONSchemaBench's home is c11), and three of c12's motivating "benchmarks" are provenance-unverified placeholders that cannot be cited, thinning the citable support specific to this candidate.

## Implementation: steps & risks

1. **Build the record generator from scratch.** No reusable repo exists, so the seeded generator over sigil-tagged, fixed-width, escaped-delimiter, and header-dialect record families is entirely new code — the load-bearing and most expensive component.
2. **Excise and replace the refuted strata.** Drop or replace the checksum stratum with a non-arithmetic comparison/lookup rule; redesign fixed-width strata around field order plus an invented padding glyph (short ≤4-char widths, distinctive pad runs) instead of character counting.
3. **Harden the distractor and normalization design.** Mandate at least two same-type distractor fields per record so semantic type-matching cannot short-circuit the sigil stratum; randomize the implied-decimal position per run (2/3/4 places) so normalization is an inducible rule rather than a fixed convention.
4. **Add the pre-freeze calibration gates.** Require the spec-dump reference prompt to clear a high per-stratum ceiling on the pinned model and seed before a stratum enters the pool, and record per-stratum reference/floor scores in the manifest so gap-recovery is falsifiable per stratum.
5. **Fix the output surface and provenance.** Stratified splits with even dialect/normalization mixes, a single-token output with a published exact-match normalizer, and pinned generator + seed metadata.

### Risks
- **From-scratch build with no anchor code** — the double-weighted reuse axis is zero, so every hour is greenfield generator and oracle work with no maintained repo to validate against.
- **Batch-level-only diagnosis** — single-token exact-match failures carry almost no per-instance signal, so reflection/gap-recovery must operate on aggregated per-stratum scores, weakening the very diagnostic loop the task is meant to exercise.
- **Prompt shortcut on the sigil stratum** — without ≥2 same-type distractors per record, a generic type-matching prompt collapses the intended headroom to near-ceiling.

### Repo review & run verification
- **Codebase:** none. No repo review file exists for this candidate, and round-2 discovery found no plausibly reusable codebase — every candidate repo was a frozen dataset, LLM-driven, or the wrong task shape.
- **Run verification:** never run-verified — there was no generator to run (I1=0).
