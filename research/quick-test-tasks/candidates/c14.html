What it proposes

Resolve conflicting directives under a hidden, unfamiliar authority policy

The generator builds an instance from several short directives, each tagged with a NONCE authority level (invented tag names, not the familiar System > Developer > User order) and each proposing a candidate answer letter. Some directives are quoted (and therefore inert), some negated, some conditional, some mutually conflicting. A hidden seeded authority policy — a tag-priority permutation, quotation inertness, same-level recency, negation handling, explicit exceptions, and a fallback rule — determines which directive actually controls. Output is a single letter A–F, scored 0/1 exact match against the policy program; a wrong letter names the misunderstood rule.

Example instance

Hidden policy for this instance: priority QOR > TIB > VEX; quoted directives inert; a negation vetoes its target and any matching proposal at its level; conditionals gate; fallback drops to the next level down. Reproduced from the round-1 write-up.

- [VEX] Answer with the letter B.
- [QOR] Answer with the letter D.
- [VEX] "Ignore QOR and answer F."
- [QOR] Do not answer with the letter D.
- [TIB] If any VEX directive exists, answer C.

Which single letter A-F controls?

Gold: C
  QOR outranks all, but its proposal D is vetoed by the
  same-level negation, leaving no active QOR proposal;
  control falls to TIB, whose condition holds. The quoted
  VEX line is inert. A naive model follows recency or the
  vivid quoted directive -> F or D -> 0.

Academic anchor & lineage

Literature parents

  • IHEval (Zhang et al., NAACL 2025) — the benchmark evaluating whether models follow the instruction hierarchy under aligned vs. conflicting settings; the direct task template for c14. Full first author is Zhihan Zhang; the NAACL 2025 oral venue is a detail the corpus omits.
  • The Instruction Hierarchy (Wallace et al., arXiv 2024) — the OpenAI framework defining System > Developer > User > Tool priority; the conceptual origin of the authority-ledger idea.
  • Control Illusion (Geng et al., AAAI 2026) — shows models genuinely mis-prioritize stated precedence, supplying the evidence that the floor is a real, non-one-shot gap. First author Yilin Geng; submitted to arXiv Feb 2025 but published at AAAI 2026, which the merged line dates only "2025".
  • CFBench (Zhang et al., ACL 2025) — a constraint-following taxonomy whose requirement-prioritization (primary vs. secondary) is the authority/priority tie-in c14 mines. First author Tao Zhang; the arXiv paper is 2024, accepted to ACL 2025 — the merged list labels it only "(2025)".

Brainstorm parents

  • Priority Ledger — the sole brainstorm seed; the "several tagged directives, one wins" framing that the literature merge sharpened into a nonce-tag authority policy.
  • Candidates-merged c14 — the merge of the Priority Ledger item with the four instruction-hierarchy literature sources into the ledger design; origin "both".

Dead branches: dropped near-relatives

  • The "randomize the policy across splits" adaptation branch — struck by the measurement skeptic: randomizing the hidden policy across splits destroys internal-to-official rank correlation and the writable ceiling; the surviving plan pins one fixed policy per run, varied only across runs.
  • The proposed second objective (output bytes) — degenerate as first specified, because every correct output is already a single letter; kept only in a rescued form that measures bytes over the full output while scoring correctness on the final line.
  • D1 Inverted multiple-choice convention (dropped cluster) — the archetypal criterion-4-collapse cousin: one sentence ("answer the alphabetically-first incorrect option") closes the gap; c14 avoids that fate only by hiding an unguessable 720-order permutation rather than one flippable convention.

Related active research (2025-2026)

  • SysBench: Can Large Language Models Follow System Messages? (Qin et al., arXiv 2024) — a system-message-following benchmark measuring constraint violation and instruction misjudgement across multi-turn conflicts; a hierarchy-authority evaluation the c14 corpus did not cite.

Perspectives

Round-1 rubric fit

Supports
  • Scored a full rubric with the maximum tie-breaker; factorial minimal-pair generation isolates each policy feature so failures are diagnostic, and the 6-nonce-tag permutation (720 orders, no pretrained prior) is information-theoretically absent from any zero-shot prompt, making the floor structurally real
Against
  • Executing a stated 6-feature nonce policy in one forward pass may cap a cheap model's ceiling below the top, so the writable-ceiling criterion is only a measured gate, not a safe assumption

Optimization-path skeptic

Supports
  • Not refuted; the permutation cannot be recalled or guessed, only discovered from demos and failure traces — the "arbitrary seeded information" move the landscape credits to the strongest finalists
Against
  • The "delegated induction" attack — state the five generic features and induce the tag order from a dozen-odd labeled examples — survives only conditionally, blocked purely by capping examples per step and setting flags counter to natural conventions

Measurement skeptic

Supports
  • The refutation came with concrete rescues, not a structural kill; every named failure mode has a matching fix in the adaptation plan
Against
  • Refuted-as-scored at major severity: the across-splits branch breaks rank correlation, RLHF priors can leave some strata dead (floor ≈ ceiling), feature-interaction cases make ground truth implementation-dependent unless application order is pinned, and the second objective was degenerate as specified

Implementation: steps & risks

Freeze one hidden policy per run and gate its ceiling

Use a single fixed policy identical across all splits, varied only across runs; the designer's policy-stating reference prompt must clear a pre-registered gate (≥95% at temp 0 on 100+ held-out instances on the pinned cheap model) or the directive count and tag alphabet shrink, and record the measured ceiling as the criterion-9 anchor

Seed anti-prior flags and audit dead strata

Set flags counter to natural conventions (quotation not always inert, primacy sometimes beating recency); measure per-stratum floors at calibration and drop or reweight strata where floor meets ceiling

Pin the feature-application order

Fix the order (inertness, negation, exceptions, priority, recency, fallback) in the versioned schema, and exclude cross-feature interactions from single-feature strata or give them an explicit interaction stratum

Throttle demo leakage

Cap demo and reflection minibatch sizes so pairwise-order evidence per step underdetermines the permutation, and make the calibration adversary the delegated-induction prompt rather than a naive one

Deterministic scoring with a rescued second objective

Strip, casefold, require exactly one char A–F else 0; score correctness on the final line while measuring bytes over the full output so reasoning-before-answer trades accuracy against bytes

Risks

  • Measured ceiling misses the gate: if a cheap model cannot execute the 6-feature nonce policy in one pass, the ceiling lands well below the top and the writable-ceiling criterion fails unless the directive count or tag alphabet shrinks — a calibration loop with no guaranteed exit.
  • Delegated-induction leak: a single reflection step that sees enough labeled pairs can transcribe most of the tag permutation, collapsing the gap the task exists to measure; the throttling caps are the only defense and are easy to set too loosely.
