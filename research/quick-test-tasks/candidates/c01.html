c01Invented-Glyph Positional Base Arithmetic

GATED
You know this as: the counterfactual non-decimal base-arithmetic probe from Reasoning or Reciting? — dressed in a per-instance invented glyph alphabet so the model cannot fall back on memorized base tables.

Gated in round 2 — refuted by both round-1 skeptic lenses at major severity, with no concrete rescue. The task is structurally attractive: a ten-line positional evaluator gives exact ground truth, per-instance random legends make it contamination-proof, and the literature documents a genuine default-prompt floor. But it is trapped between two failure modes its own design cannot escape at once. When the base and legend are stated explicitly, a single competent instruction — read the glyph-to-digit map, apply place weights — one-shots the pool and collapses the incremental gap criterion four requires. When the quirks are left implicit, there is no writable reference prompt that reaches roughly full accuracy, so there is no known ceiling to optimize toward. Compounding this, the natural inverse of encoding a decimal into the glyph system has non-unique ground truth once null-glyph and signed-digit strata are active, which breaks exact-match scoring. The gate is a round-1 verdict-health decision, so it applies regardless of the implementation score: the only fixes are structural redesigns that turn this into a different candidate, closer to c03's opaque keyed operators. The implementation total is recorded for completeness but does not lift the gate.

What it proposes

Evaluate a numeral in a per-instance invented glyph alphabet under a nonstandard positional base

A seeded generator emits one or two numerals in an invented glyph alphabet together with an inline legend (glyph→digit map) and a base spec that varies per stratum: base 9 or 11 (bases 8 and 16 are avoided as pretraining-frequent), mixed-radix, balanced or signed digits, MSB- vs LSB-first ordering, and a null-glyph position skip. The model returns a single decimal integer (or a fixed-width glyph string), scored by exact match against a roughly ten-line positional evaluator. The latent rules — legend parse, place weights, signed-digit handling, endianness, null glyph, carry variant — are stratified across the pool so no fixed one-shot instruction covers every quirk at once, and per-instance random legends make the pool contamination-proof.

Dead branches: dropped near-relatives

  • Encode direction (decimal → glyph string) — dropped as a scored surface: once null-glyph and signed-digit strata are active, the encode direction has non-unique ground truth, so exact-match scoring is undefined.
  • Bases 8 / 16 — excluded from the base spec: pretraining-frequent, so a model leans on memorized tables and the floor is too high.
  • c02 Fictional Mixed-Radix Unit Conversion — a sibling that stayed alive but was edged out on research currency; it overlaps c01/c10.
  • Cryptarithmetic (general) — folded to a hard-tail stratum only; full cryptarithmetic is combinatorial search that favors scratchpads and is noisy at temperature 0.
  • D3 TokenCounter / D9 char-level string tasks — adjacent glyph-manipulation ideas killed because letter/char counting is tokenization-bound and noisy at temp 0, threatening criterion 5.

Against

Implementation: steps & risks

Subclass the reasoning-gym generator

base_conversion ships standard-glyph base conversion with no invented legend; write a new ProceduralDataset subclass (~80-120 LOC) that samples value + base with rng = Random(seed + idx), samples an invented-glyph permutation from a nonce alphabet, and renders the source numeral under the legend

Build the oracle and strict scorer

Compute the gold via standard integer conversion mapped through the same legend; override score_answer for strict 0/1 exact match (~10 LOC), since the base class gives substring partial credit

Stratify the latent rules

Expose base value, endianness, null-glyph skip, and signed/balanced-digit family as config fields so no single instruction sentence covers the pool; avoid bases 8/16; cap demo coverage so few-shot cannot leak the whole convention set

(Never reached) rescue gates

A pass would require a red-team check that no fixed one-shot prompt matches the ceiling and a resolution of the encode-direction non-uniqueness — but these are structural redesigns, so the candidate is gated before this step

Risks

  • One-shot collapse: an explicit-spec prompt reaches the ceiling in a single call, erasing the incremental gap (the major optimization-path refutation).
  • No known ceiling under implicit quirks: without a writable roughly-full-accuracy reference prompt, the measurement story has no anchor (the major measurement refutation).
  • Non-unique encode ground truth: the decimal→glyph direction admits multiple correct strings once null/signed strata are on, so it cannot be scored by exact match.
  • Decimal round-trip shortcut: CoT-capable models can "convert to base-10 and back"; invented glyphs and bounded output are the only mitigations, and they only partly work.
  • Shared-logic oracle: the reasoning-gym oracle reuses the generator's own conversion routine, so a shared bug would corrupt prompt and answer symmetrically (no independent cross-check).

Repo review & run verification

  • Codebase: open-thought/reasoning-gym (Apache-2.0) — deep-read notes at repos/open-thought-reasoning-gym.md. Seed plumbing is threaded and clean (Random(self.seed + idx) per item); the invented-glyph semantics are net-new (~250-350 LOC in our tree, 0 lines changed in reasoning-gym).
  • Run verification: partial the base_conversion generator ran on 2026-07-21, was byte-identical at seed 42 across runs, varied at seed 123, and passed ground-truth spot checks — confirming the infrastructure. The invented-glyph task layer itself was never run-verified because it is not yet written; I1 held at 2 for exactly this reason.
