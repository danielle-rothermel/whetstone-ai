Ranked #6 of 20 after the implementation-aware rescore — round-1's outright winner, still the best theoretical task in the pool. In round 1 this was the single cleanest design: a straight sheet of top marks on the rubric and the only finalist both adversarial lenses left standing with nothing worse than minor, pinning-level caveats. What moved it down was not the task but the build. Round 1's own adaptation table demands the most intricate generator in the set — a graph/nonce sampler, per-seed relation-property profiles, a unique-answer rejection sampler, MC/foil balancing, and stratified minibatches — which scores the moving-part axis at zero. On top of that sit four-plus pre-freeze calibration gates (empirical ceiling tuning, naive floor, a literal-logician baseline held below roughly half, a per-rule discoverability check, and an adversarial induce-the-conventions probe), which scores the calibration axis at zero as well. Both zeros are unique among eligible top-tier candidates. The codebase is not the problem: it rides the same run-verified prontoqa base as c18, so the reusable-codebase axis is a full mark and the skeptic-debt axis is a full mark. The saving grace is that the c18 harness is roughly four-fifths of a future c21 harness, so the backup path keeps this candidate alive at marginal cost.

What does it propose?

Relational QA over a nonce graph whose relation rules and answer words are both seeded secrets

A pinned generator builds a small graph of nonce entities connected by nonce relation symbols. Each relation symbol carries seeded, nonstandard properties fixed per generator seed: which relations are symmetric, which are transitive, an inverse mapping, a two-relation composition table, explicit-negation precedence, and closed-world handling. Answers use an invented closed vocabulary — a gender-neutral nieph, a bond- prefix for in-law-style links, a relation-distance cutoff. An instance is a few facts plus one query; the output is one or two words from the closed vocabulary or a 4-way MC letter (true / false / both / unknown), scored 0/1 exact match against a graph-traversal oracle that computes the answer under the seed's property assignments. Two independent sources of headroom stack: the model must learn the codebook and the traversal rules, and failures cleanly separate codebook misses (a standard English kin term slipped out) from traversal misses.

Example instance

One seed's conventions: tovv is transitive and non-symmetric; two tovv hops map to the codeword nieph; brell is symmetric; the closed-world default for tovv is unknown. From recommendations.md; conventions are the seeded latent rules.

Facts:
- Marn tovv Belu.
- Belu tovv Sagre.
- NOT (Sagre brell Marn).

Q1: What is Sagre to Marn? Answer with one word from the task vocabulary.
Gold: nieph  (two tovv-hops).
A naive prompt answers "grandchild" -> exact-match 0.

Q2: Does Belu tovv Marn? A (true) / B (false) / C (both) / D (unknown).
Gold: D  (tovv is not symmetric under this seed;
          closed-world default for tovv is unknown).
A naive prompt guessing symmetry answers A -> 0.

Where does it come from?

Literature parents

  • CLUTRR (Sinha et al., EMNLP 2019) — the canonical kinship/relational-reasoning benchmark: infer family relations over a graph of characters. Direct genre ancestor; notably absent from all of c21's own references, a genre-anchor gap flagged during citation verification.
  • PrOntoQA (Saparov & He, ICLR 2023) — the corpus's named adjacency: fictional-predicate synthetic reasoning with proof-step diagnostics. c21 borrows its nonce-symbol contamination-proofing. (Round-1 corpus claimed "no arXiv ID"; the ID exists — 2210.01240 — corrected here.)
  • PrOntoQA-OOD (Saparov et al., NeurIPS 2023) — OOD deductive-reasoning extension of PrOntoQA; supports the per-seed rule-composition strata.
  • StepGame (Shi et al., AAAI 2022) — multi-hop relational (spatial) QA over generated graphs, contamination-resistant; the closest structural sibling to c21's traversal oracle. Also absent from c21's references.

Brainstorm parents

  • Candidates-merged c21 — a brainstorm-origin candidate, merged from two seeds: Kinship Convention QA and Relational Micro-World. It has no "Merged from (literature)" list (that line is a dash), so its academic sources are recoverable only from the recommendations tie-breaker paragraph.
  • Tie-breaker line — names the in-context rule-learning / synthetic-ICL family it sits in: cipher-ICL, GINC/RegBench, MIR-Bench, FalsifyBench, and the LINGOLY/Linguini linguistics-olympiad family, plus direct adjacency to PrOntoQA.

Dead branches: dropped near-relatives

  • FalsifyBench — named as a c21-adjacent line in recommendations.md but unverifiable: no confident arXiv/venue match was found. Reported as unverified provenance rather than laundered into a citation.
  • c15 Opaque-Codebook Remapped-Label Classification — a same-family remapped-label sibling, gated out in round 2: both skeptic lenses refuted it at major severity with only structural-redesign rescues.
  • c05 — refuted by both skeptic lenses at major severity; its "latent rules" were textbook protocol/interpreter conventions a competent prompt engineer states one-shot, and its fixes amounted to redesigning it into something much closer to c21/c03.

What's the current research context?

  • MIR-Bench (Yan et al., arXiv 2025) — corpus tie-breaker anchor: many-shot in-context pattern/function induction, the ICL rule-induction line c21 sits in.
  • ICL CIPHERS (Fang et al., arXiv 2025) — corpus "cipher-ICL" anchor: substitution-cipher vocab remapping to force genuine ICL; parallels c21's invented closed vocabulary.
  • GINC / ICL as Bayesian Inference (Xie et al., ICLR 2022) — corpus "GINC" anchor: synthetic HMM-generated ICL dataset; foundational synthetic-ICL substrate cited for c21's line.
  • Linguini (Sanchez et al., arXiv 2024) — corpus LINGOLY/Linguini anchor: language-agnostic olympiad reasoning, inferring the rules of unseen systems in-context; kin to the invented-vocab framing.
  • In-Context Language Learning: Architectures and Algorithms / RegBench (Akyurek et al., ICML 2024) — source of the "RegBench" regular-language ICL setup the corpus names (a usage name; the paper frames it as ICLL over random-finite-automaton regular languages). The synthetic-automaton in-context rule-inference substrate c21 generalizes to relations.
  • Rashid: A Cipher-Based Framework for Exploring In-Context Language Learning (Bafna et al., arXiv 2026) — 2026 cipher/invented-language ICL framework; constructs unseen languages via reversible ciphers. Verified real despite the unusual 2026-03 ID; the corpus lists it but did not verify provenance.

How do different lenses see it?

Round-1 rubric fit

Supports
  • Straight top marks on all fourteen criteria — the hidden rulebook is arbitrary seeded information no zero-shot prompt can contain, so the low-floor/incremental-gap criterion is structural rather than tuned; roughly nine independent latent strata give the finest-grained ladder in the pool.
Against
  • Even at a perfect scorecard, a best-effort "uninterpreted-symbols, closed-world literal logician" prompt could land mid-range if the hidden-rule strata are under-weighted — a floor that must be pinned down by weighting, not assumed.

Optimization-path skeptic

Supports
  • Not refuted — only minor caveats. The per-seed property assignments and invented codebook cannot be recalled or guessed and, with only a few facts per instance, are under-determined from any single instance, so "infer the rules yourself" prompts cannot shortcut the ladder.
Against
  • In-context rule induction via large demo sets is a real leakage path; the plan caps few-shot demo counts and adversarially tests an induce-the-conventions prompt precisely because that path exists.

Measurement skeptic

Supports
  • Not refuted — only minor caveats. Failures are diagnostic: codebook misses are cleanly distinguishable from traversal misses, and within traversal the violated rule is identifiable from expected-vs-got.
Against
  • Some rules (composition table, distance cutoff) may be practically non-inducible from 0/1 feedback, stalling every optimizer at the same sub-ceiling; multiple-valid-answer instances and an incoherent "both" label are latent generator bugs until rejection sampling removes them.

Implementation lens (round 2)

Supports
  • Rides the same run-verified prontoqa base as c18 — the reusable-codebase axis is a full mark, skeptic-debt is verification-only (a full mark), and the c18 harness is roughly four-fifths of a future c21 harness, so the option stays cheap.
Against
  • The moving-part axis and the calibration axis both score zero — the most intricate generator in the set plus four-plus pre-freeze gates. Both zeros are unique among eligible top-tier candidates and are what dropped it from first to sixth.

Research currency

Supports
  • Sits squarely in the active in-context rule-learning / synthetic-ICL and rule-induction lines (cipher-ICL, GINC/RegBench, MIR-Bench, LINGOLY/Linguini) and is directly adjacent to PrOntoQA's fictional-symbol methodology; it could grow into a standalone optimizer-comparison benchmark.
Against
  • No flagship live leaderboard to anchor to — the tie-breaker's only noted gap — and its best-known genre anchors (CLUTRR, StepGame) are missing from its own reference set.

How would we build it, and what could go wrong?

Build the two pinned artifacts

A graph sampler (nonce entities plus per-seed relation-property profiles) and a traversal oracle computing exact answers; pin generator version, dataset manifest, and disjoint split identities. On the prontoqa base this maps to --OOD --deduction-rule Composed --proofs-only --ordering random plus --disjoint-concept-names for the invented vocab.

Cap hop depth and balance outcomes

Tune hop depth down until the full-rule reference prompt reaches roughly 100%; balance the four MC outcomes and the closed-vocab answer distribution so constant-guess baselines pin low (25% MC floor).

Tag strata and reject non-unique answers

Tag every instance with its governing latent-rule stratum for failure attribution and stratified aggregation; rejection-sample any instance whose rule-closure admits more than one valid closed-vocab answer, and define "both" as arising only for relations lacking negation-precedence.

Run the pre-freeze gates

Store the reference ~100% prompt, the naive baseline, and a literal-logician baseline weighted to score at or below half; run a per-rule discoverability check (drop rules a reflection LM cannot state from failure traces); then pre-register a falsifiable success criterion.

Normalize and score strictly

Lowercase, strip whitespace, canonicalize hyphenated bond- terms; unparseable / off-vocabulary / empty outputs are decisive zeros. Add mean output UTF-8 bytes as a distinct second objective for two-objective coverage.

Risks

  • The generator is the pool's most intricate: graph/nonce sampler + relation-property profiles + unique-answer rejection sampler + MC/foil balance + stratified minibatches — the moving-part axis scores zero, so the effort is front-loaded and error-prone.
  • Four-plus pre-freeze calibration gates (empirical ceiling, naive floor, literal-logician baseline, per-rule discoverability, adversarial induce-conventions) mean the calibration axis scores zero; if discoverability fails on the composition table or distance cutoff, those strata must be shrunk or dropped before freeze.
  • On the prontoqa base, the Composed/relational path is exactly where the reachable --ordering postorder/preorder crash lives; the driver must force --ordering random (or apply the one-line list(reversed(theory)) fix), and the generator must run from the repo root for bad_patterns.txt.

Repo review & run verification

  • Codebase: asaparov/prontoqa (Apache-2.0) — deep-read notes at repos/asaparov-prontoqa.md.
  • Run verification: partial — the prontoqa generator was run in a fresh venv — same-seed byte-identical, different-seed varies, three ground-truth hand-checks correct — but that confirms the shared base, not c21's own relational/invented-vocab strata, rejection sampler, or calibration gates, none of which have been built or run.
