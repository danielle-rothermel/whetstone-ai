## Verdict

**Gated out in the round-2 rescore.** This was a round-1 finalist on the raw rubric — one of the seven top-tier scorers — and its implementation story is middling but not disqualifying on its own. ## What it proposes

### Task: classify short texts into opaque codes under a hidden multi-rule feature-to-code function

A pinned generator produces short template-filled texts (reviews or event sentences) and labels each with an **opaque code** (`ALPHA`..`DELTA`, or compositional tags like `#mix.cal`) assigned by a deterministic **feature-to-code function** with four or more interacting rules: a base **polarity map**, a **sarcasm-inversion** trigger, a **grudging-praise** trigger (cued by phrases like "finally" or "after N tries"), a **neutral-fact carve-out**, an optional **weekday suffix**, and a fixed **precedence order** among them. The model must emit exactly one code token, scored 0/1 by exact match. The point is mechanistic: a default prompt that follows the pretrained sentiment prior scores near floor — the documented flipped-label behavior — and each latent rule the prompt captures lifts its own stratum, so the score ladder is meant to be incremental. Seeded slot-filling generation keeps the pool contamination-proof.

### Example instance

*Illustrative, constructed faithfully from the merged summary (the corpus has no run-verified seeded instance for c15). One seed's conventions: base positive maps to `ALPHA`; sarcasm inverts; grudging-praise outranks sarcasm; codes are opaque so no prior helps.*

```
Text: "Finally, after three broken replacements, this one actually works."

Latent features: grudging-praise trigger fires ("finally", "after N tries");
surface sentiment reads positive; grudging-praise outranks the base map.
Gold: GAMMA  (grudging-praise stratum, per this seed's precedence)

Naive prompt ("classify the sentiment as positive/negative"):
answers "positive" -> not a code token -> exact-match 0.
A one-mapping prompt that only knows base polarity answers ALPHA -> 0.
```

## Academic anchor & lineage

### Literature parents

- [Larger language models do in-context learning differently (Wei et al., arXiv 2023)](https://arxiv.org/abs/2303.03846) — the canonical flipped-label / SUL-ICL finding; overriding semantic priors is emergent with scale, and this is the mechanism that makes the default prompt score near floor with a known ceiling. c15's core citation.
- [Symbol tuning improves in-context learning (Wei et al., EMNLP 2023)](https://arxiv.org/abs/2305.08298) — the arbitrary-symbol (foo/bar, ALPHA/BETA) label convention; this is the opaque-codebook label space c15 borrows.
- [Rectifying Demonstration Shortcut in In-Context Learning (Jang et al., NAACL 2024)](https://arxiv.org/abs/2403.09488) — the demonstration-shortcut / label-bias evaluation; the template for instances where the prior-driven answer is wrong and only the stated mapping is right. (The c15 corpus attributes this to "Kang et al." at research-classification-under-nonstandard-remappe.md line 150; the verified first author is Joonwon Jang — corrected here. The corpus itself flagged the author list as uncertain.)

### Brainstorm parents

- [Candidates-merged c15](../candidates-merged.md#c15) is a **both-origin** candidate, merged from two brainstorm seeds — *Contrarian Sentiment Codebook* and *Cipher-Category Tagger (symbol semantics remap)* under the brainstorm's Theme F — plus the literature pillars above. The generator design (short texts, opaque codes, layered triggers with precedence) is the brainstorm's contrarian-codebook shape made concrete.
- [Brainstorm Theme F — Classification under nonstandard / remapped label semantics](../brainstorm.html#theme-f) — the seed theme this candidate instantiates: contrarian sentiment codebooks, cipher-category tagging, priority/override policies.

Its cheap single-call MC format is inherited by c13/c15 as a synthetic reframe, but the datasets themselves are format-template-only, not citable sources for this candidate.

## Related active research (2025-2026)

- [Semantic Anchors: Why Small LLMs Cannot Flip Their Labels (Krishna Kumar, arXiv 2025)](https://arxiv.org/abs/2511.21038) — cautionary calibration: 1–12B models show a zero semantic-override rate under inversion, so c15's floor may be immovable on a cheap model, which would break the incremental-gap criterion for reasons unrelated to prompt quality.
- [In-Context Fixation: When Demonstrated Labels Override Semantics (Liu, arXiv 2026)](https://arxiv.org/abs/2605.08295) — nonsense-token label vocabulary and a format-vs-content decomposition mapping onto independent latent rules; documents small-model collapse mirroring Semantic Anchors. A very recent (May 2026) work, not an established reference.
- [Belief Dynamics Reveal the Dual Nature of In-Context Learning and Activation Steering (Bigelow et al., arXiv 2025)](https://arxiv.org/abs/2511.00617) — a Bayesian account of steering vs ICL over latent-concept beliefs; grounds whether prompts can drive a model between the standard and the flipped label spaces c15 depends on.

## Perspectives

### Round-1 rubric fit
**Supports:** Scored in the top tier on the raw rubric — best simultaneous alignment with the core criteria in its domain, since the opaque codes defeat the prior (low floor, known ceiling), the layered rules give an incremental decomposable ladder, output is one token, and demonstrations of the triggers measurably help.
**Against:** The floor and ceiling depend on the flipped-label mechanism being available on the chosen model at all; on a small model the whole ladder can sit at floor regardless of prompt.

### Optimization-path skeptic
**Supports:** In principle the layered triggers plus precedence give more than a bare flip, so the score is not a single guess away.
**Against:** Refuted at major severity — the rule *family* is famous published literature, so a meta-prompt of the form "here are the rule types, induce the bindings from the demos" recovers the whole codebook in one shot; the only rescue (a per-instance key or compositional labels so no paragraph captures the mapping) is a structural redesign.

### Measurement skeptic
**Supports:** Failures are diagnostic in principle — the code emitted vs the gold code separates which stratum (base map, sarcasm, grudging-praise, carve-out) the prompt missed.
**Against:** Refuted at major severity alongside the optimization lens; and per the Semantic Anchors / In-Context Fixation caution the ladder can be non-monotonic or collapsed on cheap models, so a flat result would not distinguish optimizer quality from model incapacity.

### Implementation lens (round 2)
**Supports:** The oracle is trivial and genuinely independent — ground truth is the code the feature-to-code function assigns, checked by exact string match; a permissive MIT repo exists with a reusable ~5-line remap idiom.
**Against:** The gate zeroes the skeptic-debt axis, which alone caps the total; the one candidate repo, xdwang0726/ICL_LL, is a frozen-dataset pipeline (samples real HF datasets, no single-LLM-call scoring, seeds set at four-plus sites, codebook regenerated at eval time), so serving c15 is a rewrite of sampling, oracle-serialization, and scoring — the reuse axis is held to its floor.

### Research currency
**Supports:** Sits directly on an active 2025-2026 line — flipped-label / symbol-tuning ICL, with fresh 2025-2026 work (Semantic Anchors, In-Context Fixation, Belief Dynamics) still probing exactly the override behavior it exploits.
**Against:** That same currency is the refutation — the rule family is *so* well-known that inducing it from demos is a solved meta-prompt, which is what one-shots the task.

## Implementation: steps & risks

*Note: c15 is gated and not on the adoption path; these steps describe what a from-scratch build would require, distilled from the round-2 material. The candidate was never run-verified.*

1. **Confirm the model can flip labels at all.** Before any generator work, pilot the chosen cheap model on a bare inversion to confirm a nonzero override rate; per Semantic Anchors a 1–12B model may sit at floor for every prompt, which would make the ladder unmeasurable.
2. **Reimplement the generator (do not adapt the repo).** Write a self-contained input source (drop the HF `load_dataset` path entirely or pin a small local corpus), a single threaded `np.random.Generator(seed)` driving text fill, codebook draw, and strata, an invented/nonce code vocabulary, and per-instance latent-rule stratum tags.
3. **Persist the codebook with the instances.** The reuse repo regenerates the remap at eval time, decoupling generation from scoring; the ground-truth code and the mapping must be serialized with each instance so scoring is deterministic.
4. **Write single-call scoring glue.** Replace the repo's per-option LM-loss argmin (GPT2/GPT-J, Macro-F1) with one LLM call and a strict 0/1 exact-match scorer over the code token; unparseable or off-vocabulary outputs are decisive zeros.
5. **Redesign to survive the round-1 refutation, or do not ship.** Add four-plus interacting rules with precedence *and* a per-instance key or compositional labels so no single paragraph (and no "induce the rules from demos" meta-prompt) can capture the full mapping — this is the structural change the gate demands, and it converts c15 into a different candidate.

**Risks:**
- Floor immovability on cheap models — Semantic Anchors and In-Context Fixation both show ≤~12B models with a zero override rate or pathological collapse under label inversion, which kills the incremental-gap criterion regardless of prompt quality.
- One-shot guessability of the rule family — the load-bearing refutation; the codebook is famous flipped-label literature, so a meta-prompt inducing bindings from demonstrations closes the gap without a genuine ladder unless a per-instance key or compositional labels are added.
- Frozen-dataset repo dependency — the only candidate codebase synthesizes nothing; instances come from downloaded real HF datasets, so it is not cleanly re-seedable and the remap is regenerated at eval time, coupling generation and scoring nondeterministically.

**Repo review & run verification:**
- Codebase: [xdwang0726/ICL_LL](https://github.com/xdwang0726/ICL_LL) (MIT, Copyright 2022 Vector Institute) — deep-read notes at [../repos/xdwang0726-icl-ll.md](../repos/xdwang0726-icl-ll.md).
- Run verification: **never run-verified.** The repo ships no unit tests for the generator or oracle (its `test.py` is an evaluation driver), seeds are set at four-plus independent sites with a bare unseeded `random.randint` on the module surface, and the codebook is regenerated at eval time rather than persisted — nothing runnable exists to confirm same-seed determinism. c15's own adapted design was never built or run.
