What it proposes

Task: advance a position in an invented calendar of coprime cycles by per-cycle modular counting

A seeded generator builds an invented calendar of two or three coprime named cycles (Tzolk'in-style but randomized), plus a clock variant with positions labeled one through m, forward-only subtraction, and sticky-move rules. An instance presents a current tuple or clock position and an instruction to advance N; the model returns a canonical tuple string or a single position, scored 0/1 by exact match against per-cycle modular arithmetic. The latent rules that make the pool a ladder are: the moduli are independent with no shared carry, labels are 1-indexed rather than 0-indexed, one cycle carries a skip or leap rule, some moves are sticky or ignored, and the output has a canonical format. Randomized cycle lengths and names per run make the pool contamination-proof, and — the load-bearing property — a wrong-cycle answer isolates which single modulus rule broke. Quirks are stratified so no single instruction sentence covers the whole pool.

Example instance

Illustrative, constructed faithfully from the merged summary (the corpus ships no run-verified seeded c10 instance — no repository implements the invented calendar). One seed's conventions: three coprime cycles, all 1-indexed, one cycle with an every-other advance.

Calendar cycles (this seed): Vel = 1..13, Som = 1..7, Kir = 1..11.
Rule: Kir only advances on even-numbered steps (skip on odd steps).
All cycles are 1-indexed and carry independently (no shared carry).

Current: (Vel 12, Som 6, Kir 9).  Advance 5 days.

Per-cycle modular arithmetic (1-indexed):
  Vel: ((12-1 + 5) mod 13) + 1 = 4
  Som: ((6-1  + 5) mod 7 ) + 1 = 4
  Kir: advances on steps 2 and 4 of the 5 -> +2 -> ((9-1 + 2) mod 11)+1 = 10
Gold: (Vel 4, Som 4, Kir 10)

Naive miss: a model applying a shared decimal carry, 0-indexing, or
advancing Kir on all 5 steps returns the wrong cycle's value — and the
error names exactly which modulus rule it broke.

Implementation: steps & risks

Vendor reasoning-gym, author a new generator

Consume reasoning-gym as a dependency (no repo edits) and subclass ProceduralDataset in our tree; config carries sampled pairwise-coprime cycle lengths (gcd == 1), an offset/naming convention, and the latent-rule stratum id

Copy the seed idiom, write an independent modular oracle

In __getitem__ reuse the rng = Random(seed + idx) pattern from calendar_arithmetic.py:129; sample a start tuple and an advance N; compute the answer by fresh per-cycle modular arithmetic with 1-indexing and leap/skip adjustment — never stdlib datetime

Override scoring to strict 0/1 and add strata

Override score_answer for single-token exact match (~5-10 LOC), bypassing the shipped partial-credit calendar/time scorers; optionally add a BaseCurriculum subclass exposing cycle-length ranges and convention families as strata

Gate before freeze

Clear the two-to-three pre-freeze gates: verify the interacting quirks stop the modular-arithmetic prior from collapsing the gap one-shot, and stratify the arbitrary/secret conventions so no single prompt covers the pool

Risks

  • The invented interlocking-cycle semantics are net-new logic — reasoning-gym ships only real-Gregorian tasks — so this is write-the-task, not config plus glue, and the reuse score is capped accordingly.
  • Modular arithmetic is a strong pretrained prior; if the quirks are too few or too guessable a one-shot instruction closes the gap, collapsing the incremental ladder (the adaptation note's central caution, never stress-tested by a skeptic).
  • The shipped score_answer on both calendar tasks awards graded/continuous credit and the base default gives substring credit — reusing any of them would silently break the strict 0/1 requirement.
  • The reusable reasoning-gym infra is repo-run-verified (carried over from c24), but the candidate-specific interlocking-cycle task logic here is desk-estimated, not measured behavior of a built generator.
