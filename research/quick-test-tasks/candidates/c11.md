# c11 · House-Convention Canonicalizer

**Status: RANKED #8 of 20 (round-2 implementation-aware rescore).**

You know this as: **canonical-JSON / house-style normalization** — a deterministic serializer whose invented house rules deviate from every public standard (RFC 8785 JCS, slugify, ISO dates), scored by whole-string exact match.

## Verdict

**Officially ranked #8 of 20 after the implementation-aware rescore, a slip from its round-1 28-tier finalist standing.** The candidate's distinctive strength is project alignment: canonical-JSON house rules directly rehearse the dr-serialize intuitions the main experiment cares about, a transfer bonus no other finalist carries. Its generator story is genuinely reusable — json-schema-faker is a clean, seed-threaded instance producer that ran green in a fresh environment (519 of 519 tests) — but the reusable half is only the input generator; the canonicalizer and its oracle are new code we author, which is why the codebase cell settles at reusable-but-incomplete rather than drop-in glue. The rescore also inherits round 1's live obligations: the merged rule set must go on a diet (the check-digit and long-list-sort strata are things a small model cannot execute in one constrained forward pass, making the true ceiling both lower and unknowable), the rule tables must stay wider than any demo set or GEPA minibatch to block full-coverage copy-from-context, and the second objective is provably degenerate — correct canonical strings have fixed length, so output-bytes validates single-objective optimization plus two-objective plumbing only. Those are bounded fixes, but they are why it sits below the run-verified I1=3 generators rather than beside them.

## What it proposes

### Task: normalize a messy value to one canonical string under invented house rules

The generator picks ONE deterministic normalization pipeline — canonical JSON or slug are the preferred families — governed by five to six **invented house rules** that deliberately deviate from every public standard: length-then-codepoint key sort, a bespoke float and escape policy, invented month abbreviations and grouping tables, a custom slug separator and stopword list. Each **instance** is a messy input value; the model must emit the **canonical string**; scoring is 0/1 **whole-string exact match** against a roughly forty-line **reference function** that *is* the ceiling definition. Every rule owns a **stratum** of instances, so pool accuracy climbs rule by rule and the expected/got diff names precisely which rule was violated. Random inputs give an infinite, contamination-proof pool, and because the conventions are invented they are not one-shot guessable.

### Example instance

Illustrative, from the round-1 corpus (slug family; house rules: lowercase; token separator is `~` not `-`; drop words in an invented stopword list {the, of, an}; ordinals map through an invented table so `2nd` → `twyx`; keep at most 4 tokens, truncating from the end):

```
Input title:  The Quick   Brown Fox (2nd Edition)!
Gold:         quick~brown~fox~twyx

Naive slugify emits `the-quick-brown-fox-2nd-edition` → 0.
Each wrong rule (kept "the", used `-`, missed `twyx`, no truncation)
is separately visible in the diff.
```

## Academic anchor & lineage

### Literature parents

- [RFC 8785 JSON Canonicalization Scheme / JCS](https://www.rfc-editor.org/info/rfc8785) (Rundgren, Jordan & Erdtman, IETF Informational 2020) — the named task donor: canonical JSON is the generatable task, and c11 invents house rules that deliberately deviate from JCS rather than reproduce it.
- [PolyNorm: Few-Shot LLM-Based Text Normalization for TTS](https://arxiv.org/abs/2511.03080) (Wong et al., arXiv 2025) — evidence that date/phone/number normalization is a live LLM few-shot task, cited in the "PolyNorm 2025" merged-from entry (the slugify/checksum "family" itself is practitioner/standards material — python-slugify, Luhn/ISBN-13/IBAN mod-97 — not a single academic paper).
- [StructEval](https://arxiv.org/abs/2505.20139) (Yang et al., arXiv 2025) — its non-renderable slice (JSON/YAML/CSV/TOML) supplies the format taxonomy and below-ceiling floor evidence.
- [JSONSchemaBench](https://arxiv.org/abs/2501.10868) (Geng et al., arXiv 2025) — the schema-pool donor: a graded schema corpus for calibrating canonicalization difficulty, contamination-proofed by regeneration.
- [IFEval](https://arxiv.org/abs/2311.07911) (Zhou et al., arXiv 2023) — verifier-library design donor: ready-made deterministic exact checkers and the verifiable-instruction paradigm c11's scoring reuses.
- [EDC: Extract, Define, Canonicalize](https://arxiv.org/abs/2404.03868) (Zhang & Soh, EMNLP 2024) — a canonicalize-step reference for the domain; primarily c12's parent, but it anchors the canonicalization framing c11 shares.

## Perspectives

### Round-1 rubric fit
- **Supports:** Scored a full 28 — infinite contamination-proof pool, a reference function that *is* the ceiling definition, and strict no-partial-credit scoring (one wrong rule fails the whole example).
- **Against:** The tie-breaker is strong but derivative — it rides the verifiable-instruction wave rather than opening one.

### Optimization-path skeptic
- **Supports:** Not refuted (major caveats survived) — invented conventions are not one-shot specifiable, and with ≥6 stratified rules the gap accretes rule by rule across GEPA iterations.
- **Against:** The plan's own "demo sets covering all invented tables" step manufactures its strongest attack: full-coverage demos turn lookup rules into copy-from-context, and table-width vs. minibatch-width is never pinned, so one reflection pass could transcribe every rule visible in a batch.

### Measurement skeptic
- **Supports:** Refuted-as-specified but rescued by a rule-set change — restricting rules to single-pass-executable transformations restores a knowable ceiling.
- **Against:** As specified the check digit, long-list sorts, and bespoke float formatting push the true ceiling to a 70–90% band unknowable in advance, with residual slips landing exactly where temp-0 provider nondeterminism flips tokens; the output-bytes second objective is degenerate.

### Implementation lens (round 2)
- **Supports:** The generator half is genuinely reusable — json-schema-faker is seed-threaded, byte-deterministic on a fixed seed, and ran 519/519 tests green in a fresh environment; the rfc8785.py library offers a ready-made canonical-JSON oracle to fork or post-process.
- **Against:** Neither repo supplies the actual task substance — the canonicalizer plus oracle (the invented house rules) is roughly 200–350 lines of new logic, so the diff is task-logic-plus-glue, not config-plus-glue, which caps the codebase cell at 2.

### Research currency
- **Supports:** Text normalization and verifiable instruction-following are both active 2025–2026 waves (PolyNorm, IFBench, StructEval), and the dr-serialize alignment is a project-specific bonus no other candidate has.
- **Against:** No skeptic pass examined this candidate below the 28 tier — the finer-grained adversarial lenses ran on the round-1 finalists as a group, and no dedicated currency re-audit was recorded past the tie-breaker line.

## Implementation: steps & risks

### Steps

1. **Put the rule set on a diet.** Restrict rules to single-pass-executable transformations — table lookups (invented month/stopword/ordinal tables), character-class rewrites, fixed replacements, whitespace collapsing, truncation policy, sorts of ≤3 short keys. Drop the arithmetic check digit and long-list sorts (or reduce the check digit to a fully enumerable ≤3-digit lookup). Keep the rule count ≤6.
2. **Enforce table-width > coverage.** Table-valued rules get more entries (12–20) than any demo set or GEPA minibatch (cap minibatch ~5; cap demo sets below full table coverage), so rules must accrete over iterations. Strike "cover all cases" from demo curation.
3. **Make the generator adversarial per instance.** Every instance actively distinguishes a house rule from its public standard (reject inputs where they coincide); global formatting rules form an explicit first-rung stratum; ≥2 tasks per stratum in every internal subset. Reuse json-schema-faker as the seeded input producer (pin `minDateTime`/`maxDateTime` or avoid date formats to sidestep the one wall-clock nondeterminism source).
4. **Build the canonicalizer + oracle as new external code.** Write the ~150–250-line deterministic house-rule pipeline that is both the reference the model reproduces and the ground truth; optionally fork rfc8785.py (~200 lines) and edit its escape table, key-sort comparator, and float format for the canonical-JSON family.
5. **Pre-freeze certification.** Reference prompt ≥95% exact match with three agreeing temp-0 repeats on the pinned split; derive floor/gap targets from measured numbers; freeze exactly one trivial normalization (strip surrounding whitespace/code fences) in the Eval Config. Run the "infer every rule from the examples" one-shot with the largest permitted demo set and treat >50% as a rule-mix bug. Document that the output-bytes objective is degenerate here.

### Risks

- **Ceiling unknowability if the diet is skipped.** Leaving computation-heavy rules in place makes the true ceiling a 70–90% band no one can predict in advance, with residual failures indistinguishable from temp-0 provider nondeterminism — the measurement lens's refutation.
- **Full-coverage demo leakage.** If demo sets or minibatches are allowed to cover every table entry, lookup rules degrade into copy-from-context and the gap the task is meant to measure collapses; table-width policing is the only guard.
- **Degenerate second objective.** Output-bytes is fixed for correct canonical strings, so this candidate validates single-objective optimization plus two-objective plumbing only — a real limitation versus c21/c14 that must be declared per the rubric callout.
