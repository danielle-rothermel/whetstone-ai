Opaque-Codebook Classification is gated — its own rule family one-shots it

GATED

Flipped-label ICL: short texts classified into opaque codes by a hidden feature-to-code rulebook.

whetstone-ai · quick-test-tasks · round-2 rescore

Gated out in the round-2 rescore — refuted in round 1 without a concrete rescue, ineligible regardless of implementation score. This was a round-1 finalist on the raw rubric — one of the seven top-tier scorers — and its implementation story is middling but not disqualifying on its own. What removes it from contention is a health gate, not a score: both adversarial skeptic lenses refuted its measurement story at major severity in round 1, and the only fixes on offer were structural redesigns that change what the task is. The specific refutation is that its rule family — base polarity map, sarcasm inversion, grudging-praise trigger, neutral-fact carve-out, precedence order — is the famous flipped-label literature, so a single meta-prompt that says "here are the rule types, induce the bindings from the demonstrations" one-shots the whole ladder, failing the not-one-shot-guessable criterion. Under the round-2 rule that a refuted-without-rescue candidate cannot be recommended whatever its build cost, it appears here for completeness, unranked. Its implementation total is shown in parentheses to signal it was computed but does not rank; the round-2 gate zeroes the skeptic-debt axis (the refutation is unpriceable without a redesign), which is what pins the total low.

What does it actually ask the model to do?

What it proposes

Classify short texts into opaque codes under a hidden multi-rule feature-to-code function

A pinned generator produces short template-filled texts (reviews or event sentences) and labels each with an opaque code (ALPHA..DELTA, or compositional tags like #mix.cal) assigned by a deterministic feature-to-code function with four or more interacting rules: a base polarity map, a sarcasm-inversion trigger, a grudging-praise trigger (cued by "finally" or "after N tries"), a neutral-fact carve-out, an optional weekday suffix, and a fixed precedence order among them. The model must emit exactly one code token, scored 0/1 by exact match. The point is mechanistic: a default prompt following the pretrained sentiment prior scores near floor — the documented flipped-label behavior — and each latent rule the prompt captures lifts its own stratum, so the score ladder is meant to be incremental. Seeded slot-filling generation keeps the pool contamination-proof.

Where does the design come from?

Academic anchor & lineage

Literature parents

  • Larger language models do in-context learning differently (Wei et al., arXiv 2023) — the canonical flipped-label / SUL-ICL finding; overriding semantic priors is emergent with scale, and this is the mechanism that makes the default prompt score near floor with a known ceiling. c15's core citation.
  • Symbol tuning improves in-context learning (Wei et al., EMNLP 2023) — the arbitrary-symbol (foo/bar, ALPHA/BETA) label convention; this is the opaque-codebook label space c15 borrows.
  • Rectifying Demonstration Shortcut in In-Context Learning (Jang et al., NAACL 2024) — the demonstration-shortcut / label-bias evaluation; the template for instances where the prior-driven answer is wrong and only the stated mapping is right. The c15 corpus attributes this to "Kang et al."; the verified first author is Joonwon Jang — corrected here. The corpus itself flagged its author list as uncertain.

Brainstorm parents

  • Candidates-merged c15 — a both-origin candidate, merged from two brainstorm seeds (Contrarian Sentiment Codebook and Cipher-Category Tagger, under Theme F) plus the literature pillars. The generator design — short texts, opaque codes, layered triggers with precedence — is the brainstorm's contrarian-codebook shape made concrete.
  • Brainstorm Theme F — Classification under nonstandard / remapped label semantics — the seed theme this candidate instantiates: contrarian sentiment codebooks, cipher-category tagging, priority/override policies.

Dead branches: dropped near-relatives

  • D1 · Inverted multiple-choice convention — both the brainstorm and the literature flag HIGH one-shot risk; a single sentence ("answer the alphabetically-first incorrect option") closes the gap and fails the not-one-shot-guessable criterion. Its remapped-label ground is already covered by c15; usable at most as a plumbing check.
  • D7 · Contaminated classification templates (Ethos / Liar / SST-2 / Subj / TREC / AG-News) — a single latent construct, contaminated, subjective labels; the cheap single-call MC format is inherited by c13/c15 as a synthetic reframe, but the datasets are format-template-only, not citable sources for this candidate.
  • "Contrarian-codebook synthetic family" — listed in the merged entry's literature line but is not a paper; it is brainstorm-native (research file Candidate 12: "Not from a single paper"). Motivation only, not a citable academic source.

Is the surrounding research area still active?

Related active research (2025–2026)
  • Semantic Anchors: Why Small LLMs Cannot Flip Their Labels (Krishna Kumar, arXiv 2025) — cautionary calibration: 1–12B models show a zero semantic-override rate under inversion, so c15's floor may be immovable on a cheap model, which would break the incremental-gap criterion for reasons unrelated to prompt quality.
  • In-Context Fixation: When Demonstrated Labels Override Semantics (Liu, arXiv 2026) — nonsense-token label vocabulary and a format-vs-content decomposition mapping onto independent latent rules; documents small-model collapse mirroring Semantic Anchors. A very recent (May 2026) work, not an established reference.
  • Belief Dynamics Reveal the Dual Nature of In-Context Learning and Activation Steering (Bigelow et al., arXiv 2025) — a Bayesian account of steering vs ICL over latent-concept beliefs; grounds whether prompts can drive a model between the standard and flipped label spaces c15 depends on.

Does the case hold up from every angle?

Perspectives

Round-1 rubric fit

Supports
  • Scored in the top tier on the raw rubric — best simultaneous alignment with the core criteria in its domain, since opaque codes defeat the prior (low floor, known ceiling), layered rules give an incremental decomposable ladder, output is one token, and trigger demonstrations measurably help.
Against
  • The floor and ceiling depend on the flipped-label mechanism being available on the chosen model at all; on a small model the whole ladder can sit at floor regardless of prompt.

Optimization-path skeptic

Supports
  • In principle the layered triggers plus precedence give more than a bare flip, so the score is not a single guess away.
Against
  • Refuted at major severity — the rule family is famous published literature, so a "here are the rule types, induce the bindings from the demos" meta-prompt recovers the codebook in one shot; the only rescue (a per-instance key or compositional labels) is a structural redesign.

Measurement skeptic

Supports
  • Failures are diagnostic in principle — the code emitted vs the gold code separates which stratum (base map, sarcasm, grudging-praise, carve-out) the prompt missed.
Against
  • Refuted at major severity alongside the optimization lens; and per the Semantic Anchors / In-Context Fixation caution the ladder can be non-monotonic or collapsed on cheap models, so a flat result would not distinguish optimizer quality from model incapacity.

Implementation lens (round 2)

Supports
  • The oracle is trivial and genuinely independent — ground truth is the code the feature-to-code function assigns, checked by exact string match; a permissive MIT repo exists with a reusable ~5-line remap idiom.
Against
  • The gate zeroes the skeptic-debt axis, which alone caps the total; the one candidate repo is a frozen-dataset pipeline (samples real HF datasets, no single-LLM-call scoring, seeds set at four-plus sites, codebook regenerated at eval time), so serving c15 is a rewrite of sampling, oracle-serialization, and scoring — the reuse axis is held to its floor.

Research currency

Supports
  • Sits directly on an active 2025–2026 line — flipped-label / symbol-tuning ICL, with fresh work (Semantic Anchors, In-Context Fixation, Belief Dynamics) still probing exactly the override behavior it exploits.
Against
  • That same currency is the refutation — the rule family is so well-known that inducing it from demos is a solved meta-prompt, which is what one-shots the task.

What would it take to build, and what could go wrong?

Implementation: steps & risks

c15 is gated and not on the adoption path; these steps describe what a from-scratch build would require, distilled from the round-2 material. The candidate was never run-verified.

Confirm the model can flip labels at all

Before any generator work, pilot the chosen cheap model on a bare inversion to confirm a nonzero override rate; per Semantic Anchors a 1–12B model may sit at floor for every prompt, making the ladder unmeasurable.

Reimplement the generator (do not adapt the repo)

Write a self-contained input source (drop the HF load_dataset path or pin a small local corpus), a single threaded np.random.Generator(seed) driving text fill, codebook draw, and strata, an invented/nonce code vocabulary, and per-instance latent-rule stratum tags.

Persist the codebook with the instances

The reuse repo regenerates the remap at eval time, decoupling generation from scoring; the ground-truth code and the mapping must be serialized with each instance so scoring is deterministic.

Write single-call scoring glue

Replace the repo's per-option LM-loss argmin (GPT2/GPT-J, Macro-F1) with one LLM call and a strict 0/1 exact-match scorer over the code token; unparseable or off-vocabulary outputs are decisive zeros.

Redesign to survive the round-1 refutation, or do not ship

Add four-plus interacting rules with precedence and a per-instance key or compositional labels so no single paragraph (and no "induce the rules from demos" meta-prompt) can capture the full mapping — the structural change the gate demands, which converts c15 into a different candidate.

Risks

  • Floor immovability on cheap models — Semantic Anchors and In-Context Fixation both show ≤~12B models with a zero override rate or pathological collapse under label inversion, which kills the incremental-gap criterion regardless of prompt quality.
  • One-shot guessability of the rule family — the load-bearing refutation; the codebook is famous flipped-label literature, so a meta-prompt inducing bindings from demonstrations closes the gap without a genuine ladder unless a per-instance key or compositional labels are added.
  • Frozen-dataset repo dependency — the only candidate codebase synthesizes nothing; instances come from downloaded real HF datasets, so it is not cleanly re-seedable and the remap is regenerated at eval time, coupling generation and scoring nondeterministically.
