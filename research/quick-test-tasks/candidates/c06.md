## Verdict

**28-tier finalist, held at #13 after the implementation-aware rescore.** This candidate entered round 2 as one of seven verified round-1 finalists, and on the design its appeal is real: a seeded formal grammar produces strings that are either well-formed or carry exactly one injected violation, and the model answers with a single multiple-choice letter naming the violated rule class, so scoring is trivially exact and each captured constraint catches one violation class in a clean incremental ladder. ## What it proposes

### A grammar-violation classifier where the MC letter names the broken rule

A seeded generator samples strings from a randomized formal grammar composing four to five constraint types — feature-agreement pairs, nesting-depth limits, forbidden adjacencies, balanced multi-type delimiters, and one attribute-dependent depth limit — and either keeps a string well-formed or injects exactly one violation of a known class. The model outputs a single **multiple-choice letter**: A well-formed / B agreement / C nesting / D adjacency, with an **earliest-violation-by-scan-order** tie-break when a corruption could read as more than one class. Ground truth is the injected class; the score is exact letter match. Because each constraint the prompt captures catches exactly one violation class, the strata are cleanly incremental at the pool level, and the label directly names the violated rule, which makes failures maximally diagnostic. Randomized grammars per run make instances contamination-proof, and the four-way format gives a 25% guess floor. Documented variants swap the label task for next-legal-closing-bracket prediction or depth-transparent bracket-kind path queries.

### Example instance (illustrative)

*Seeded conditions: a run draws an agreement rule (tokens tagged `+hi`/`-hi` must match within a pair), a max nesting depth of two, and a forbidden adjacency (`q` may not immediately follow `p`). One violation is injected; the tie-break is earliest-by-scan-order.*

```
Grammar (this run):
  R1 agreement : paired tags must share their height feature (+hi with +hi)
  R2 nesting   : bracket depth may not exceed 2
  R3 adjacency : symbol 'q' may not immediately follow 'p'

String:  [ a+hi  [ b-hi  a+hi ] ]  p q

Question: which rule class does this string violate?
  A well-formed   B agreement   C nesting   D adjacency

Gold: D  (the 'p q' adjacency breaks R3; depth stays within 2 and the tags agree)

Naive-prompt miss: a one-shot solver that has only internalized "check the
brackets balance and nest" scans depth, sees depth 2 is fine, and answers A
well-formed — it never learned the per-run adjacency and agreement rules, so it
catches at most two of the four classes.
```

## Academic anchor & lineage

### Literature parents

- **[MLRegTest](https://arxiv.org/abs/2304.07687) (van der Poel et al., JMLR 2024)** — a benchmark of 1,800 regular languages across the subregular hierarchy scored as accept/reject membership; the core source for c06's rule-strata design, supplying the subregular families that become the violation classes.
- **[Self-Attention Networks Can Process Bounded Hierarchical Languages](https://aclanthology.org/2021.acl-long.292/) (Yao et al., ACL 2021)** — the foundational Dyck-n / bounded-depth bracket expressivity result that anchors the next-legal-bracket and nesting variants. (The "Dyck membership & next-legal-bracket" entry in candidates-merged.md is a research line, not a single paper; this is its concrete cited anchor.)
- **[Training Neural Networks as Recognizers of Formal Languages / FLaRe](https://arxiv.org/abs/2411.07107) (Butoi et al., ICLR 2025)** — releases the FLaRe benchmark across the Chomsky hierarchy; the second reusable generator donor. ### Dead branches: dropped near-relatives

- **D3 TokenCounter (distinct vowel/consonant counting)** — dropped because letter counting is tokenization-bound and StringLLM/CUTE show char-count tasks are noisy at temp 0, threatening the exact-match criterion; a live constraint on why c06's violations must be structural, not count-based.
- **D9 Char-level string-task cluster (StringLLM, CUTE, letter-counting)** — not a task, kept only as the red-flag design caveat that justifies MC/closed-vocab framing over character-level scoring in this family.
- **Pure Dyck-k balanced-membership (research candidate 10, formal-grammars doc)** — retained only as a variant, not the core: pure balanced-parenthesis membership is one skill (counting depth), so the rule decomposition is thin unless enriched with multiple bracket types and forbidden-substring constraints — which is exactly the enrichment c06 adopts.
- **Convergent cousins c21 / c03** — not dropped, but named in the round-2 reason as doing the hidden-rule-table job "more cleanly"; c06's rescue moves headroom onto hidden tables and lands on their territory.

## Perspectives

### Round-1 rubric fit
- **Supports:** Best-in-domain on output boundedness (a single MC letter) and independent-rule decomposition (each subregular constraint is one rule); scored 28, one of the seven verified finalists.
- **Against:** A 25% guess floor demands class-balanced sampling and several composed constraints so guessing is penalized; pure accept/reject would give thin diagnostics without the which-rule label.

### Optimization-path skeptic
- **Supports:** Survived the optimization-path lens with the same profile as c11/c12 — a genuinely not-one-shot-guessable, contamination-proof design puts the decisive rule knowledge outside COPRO's score-only rephrasing.
- **Against:** A convention-exploiting prompt can floor around 50-60% if the rule instantiations are mnemonic, so the fix needs a red-team gate the other finalists do not require.

### Measurement skeptic
- **Supports:** Refuted-then-rescued: the ladder can be kept honest with non-mnemonic instantiations plus label recomputation, and the exact-match MC surface makes effect sizes clean to read.
- **Against:** The rescue moves headroom off execution difficulty and onto hidden rule tables, at which point the task converges on what c21 and c03 measure more cleanly — a redundancy, not a fatal flaw.

### Implementation lens (round 2)
- **Supports:** MLRegTest ships 1,470 committed automata across 13 subregular families as ready-made strata, and both donor repos expose an independent composition/DFA membership oracle that can re-derive labels for any string (I1 2, I2 2).
- **Against:** Neither repo emits a violation taxonomy — both are binary accept/reject with undifferentiated negatives — so the entire ≥4-rule multiclass corruption layer is new code; the injected-violation isolation and few-gate structure scored weakest (I4 0), and MLRegTest's sampler is unseeded/nondeterministic.

### Research currency
- **Supports:** Tie-break 2; the subregular-membership and CFG-interpretation lines are active into 2026 (RoboGrid, learnability-of-formal-language work), keeping the anchor current.
- **Against:** Round-2 currency alone could not offset the redundancy with c21/c03 and the calibration burden; the exact-match tier of finalists simply outscored it on implementation substance.

## Implementation: steps & risks

1. **Vendor the automata and oracle, not the samplers.** Take the 1,470 committed `.fst` files and the three-line composition oracle (`pynini.acceptor(s) @ fsa` non-empty == accepted) from heinz-jeffrey/subregular-learning; the 13-family filename tag becomes the latent-rule strata. Repo edits: zero — treat it as a data + oracle donor.
2. **Write a seeded sampler.** Author `c06_gen.py` (~90 LOC, outside the repo) with a threaded `random.Random(seed)` drawing fixed-length strings over each FST's alphabet, because neither donor's sampler is controllably re-seedable.
3. **Build the missing violation taxonomy.** For each of ≥4 named rules, author a rule-specific corruption operator that breaks exactly that rule while satisfying the others (this exists in neither repo — negatives are undifferentiated), and label every string via the independent oracle rather than the tautological training labels.
4. **Balance and stratify.** Balance labels across the four classes; compose four to five constraint types including one attribute-dependent depth limit so a one-shot prompt catches only two or three of four classes; wire the earliest-violation-by-scan-order tie-break.
5. **Red-team the floor.** Gate on non-mnemonic instantiations and recompute labels, then adversarially probe for a convention-exploiting prompt that lifts the floor to 50-60%; only admit strata that clear the gate.

### Risks
- Mnemonic rule instantiations let a convention-exploiting prompt floor the task at 50-60%, collapsing the headroom unless the red-team gate is enforced per stratum.
- The rescued design shifts difficulty onto hidden rule tables and converges on c21/c03, so the marginal value over already-adopted candidates is thin.
- Heavy native dependency: the MLRegTest oracle needs `pynini==2.1.2` (conda-only OpenFst binding), the main operational risk to standing it up, unless the automata are exported to a pure-Python FSA format.
- The FLaRe donor requires a two-stage offline pipeline (`save_automaton.py` then `prepare_sampler.py`) plus an SSH-URL `rau` git dependency before any string is produced.
