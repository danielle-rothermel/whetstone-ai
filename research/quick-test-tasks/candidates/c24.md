## What it proposes

### Sort short token lists by a per-batch invented composite key, exact-match the canonical line

Each instance is three to six **tokens** or **(letter, number) pairs** — or, in one stratum, three to four small **permutations in cycle notation** with a stated composition-direction convention. A nonstandard **composite key** governs the ordering: sort by the second letter, break ties by length descending, rerank vowels after consonants, apply a per-element transform (double if even, triple if odd), and emit canonical cycle form with the smallest element leading. The model returns one comma-joined line or canonical string, scored by **exact match** against a reference sorter. Each latent rule — primary key, each tie-breaker, the rerank rule, the element transform, the separator convention — is visible as a distinct **diff signature**, so a partial prompt fails on exactly the rung it missed. Keys and directions are randomized per batch so the prompt must convey them rather than let the model guess from priors.

## Implementation: steps & risks

### Build steps

1. **Consume reasoning-gym as a dependency.** Vendor or install open-thought/reasoning-gym and reuse its `ProceduralDataset` infra and per-item `Random(seed+idx)` pattern; change zero repo lines.
2. **Write the composite-key sorting subclass.** New `ProceduralDataset` subclass, roughly 100 to 150 lines: generate records with several invented fields, sample a randomized composite key ordering (primary plus tie-breakers) per instance as the latent rule, and compute the oracle with Python `sorted(key=...)`.
3. **Override scoring to strict 0/1.** Replace the base class's substring partial-credit `score_answer` with a strict exact-match 0/1 override, so a prompt that misses one rule level scores zero on that instance.
4. **Swap in nonce vocabulary and add glue.** Bypass the bundled real-text Gutenberg corpus (`in_the_year_2889.txt`) with nonce vocab to keep instances contamination-proof; add roughly 30 to 50 lines of LLM glue for the single call and 0/1 exact-match scoring.

### Risks

- The randomized composite tie-breaker keys are net-new task logic, not a config flag — the shipped generator sorts by a single ASCII key with no per-tie-breaker diff signatures.
- A shared reference sorter couples every stratum to re-certification: a change to the oracle re-validates all key levels at once (this is the I5=1 cause).
- If the strict override is forgotten, the base class's substring partial credit silently breaks the intended 0/1 exactness.
- Off-path `pool_matrix.py` uses module-global `np.random.seed`; do not co-import it naively into a shared process, or determinism can be corrupted.

### Repo review & run verification

- **Codebase:** [open-thought/reasoning-gym](https://github.com/open-thought/reasoning-gym) (Apache-2.0) — deep-read notes at [repos/open-thought-reasoning-gym.md](../repos/open-thought-reasoning-gym.md).
- **Run verification: verified.** Ran in a fresh venv; same-seed byte-identical, different-seed varies, and all six hand-checked `word_sorting` instances matched an independent `sorted()` oracle. The invented composite-key semantics themselves are never run-verified — they are the net-new code still to be written.
