c18Depth-Controlled Synthetic Deduction (True/False/Unknown)

BACKUP
You know this as: RuleTaker / ProofWriter / PrOntoQA-style depth-binned synthetic deduction — sampled facts plus if-then rules over fictional predicates, queried for a True/False/Unknown entailment at a controlled proof depth.

Backup behind the grid-world pick, and the highest-value insurance in the pool. c18 sits second after the implementation-aware rescore because it pairs a strong task idea with the single most decisive practical fact discovered in round 2: the asaparov/prontoqa generator was run-verified end-to-end, threading --seed into both random and np.random to emit byte-identical gold-labeled JSON with no model call. That makes the entire adaptation roughly a hundred lines of subprocess-and-scoring glue with no required repo edits, so its middle-of-the-pack implementation total is dragged down almost entirely by design breadth — a second constraint-puzzle stratum and CWA-vs-Unknown pinning — rather than by any codebase doubt. The clinching reason it is the named backup rather than a lower-ranked alternate is leverage: the c18 harness is roughly four-fifths of a future c21 harness, so building it keeps round 1's theoretical winner alive at marginal cost.

What it proposes

Classify a multi-hop entailment at a dialed-in proof depth

A seeded generator regenerates RuleTaker/FLD/PrOntoQA-style theories — sampled facts plus if-then rules over fictional predicates ("every wumpus is a yumpus") — with controlled proof depth (D0-D5), distractor rules, and closed- versus open-world semantics. Each instance is a theory plus a query; the model emits True / False / Unknown, scored by exact match. Ground truth comes from the generator's proof engine, which also names the failed inference step for reflection. Latent rules are chaining depth, negation-as-failure versus explicit negation, open-world Unknown, exceptions, and quantifiers; the depth bins give both the incremental ladder and a known ceiling. A unique-solution seating-puzzle stratum covers the CSP flavor, and fresh symbols per run make it contamination-proof.

Example instance

Illustrative, hand-traced from a run-verified prontoqa output — seeded with --ontology fictional, two-hop depth, seed 12345. The nonce predicates and the negation flag come from the seed, so the surface text carries no real-world prior.

Facts & rules:
  Wren is an impus.
  Every impus is a vumpus.
  Vumpuses are not loud.
Query:   Wren is not loud.
Gold:    True     (impus -> vumpus -> not loud; a two-hop chain)

Naive-prompt miss: a model that answers from surface
plausibility, or that treats "not loud" under the wrong
world assumption, returns False/Unknown; only following
the two stated rules in order yields True.

Academic anchor & lineage

Literature parents

  • RuleTaker / Transformers as Soft Reasoners over Language (Clark, Tafjord & Richardson, IJCAI 2020) — the core template: synthetic facts plus if-then rules, depth-binned D0-D5 theories, True/False under closed-world assumption with negation-as-failure.
  • ProofWriter (Tafjord, Dalvi & Clark, Findings of ACL-IJCNLP 2021) — the RuleTaker successor adding open-world semantics with explicit Unknown; the source of the T/F/Unknown three-way label.
  • PrOntoQA / Language Models Are Greedy Reasoners (Saparov & He, ICLR 2023) — the fictional-ontology generator behind the reused codebase; contamination-resistant symbols and proof-step failure diagnosis.
  • PrOntoQA-OOD (Saparov et al., NeurIPS 2023) — the OOD depth/compositionality extension supplying the incremental-ladder and headroom evidence.
  • FLD (Morishita et al., ICML 2023) and FLD* (Morishita et al., NeurIPS 2024) — a principled complete-rule-set generator with proved/disproved/unknown labels under controllable depth and distractors.
  • LogicNLI (Tian et al., EMNLP 2021) — controlled FOL-NLI with a small AST inventory, depth up to five; an abstract/templated regeneration model.
  • SATBench (Wei et al., EMNLP 2025) — the recent hard tail: a SAT/CNF-to-puzzle generator with a solver oracle and documented large headroom.
  • RuleBERT (Saeed, Ahmadi, Nakov & Papotti, EMNLP 2021) — the soft-rule stratum; and FOLIO (Han et al., EMNLP 2024) held as a style reference only (human-written, ~1.4k items, not synthetically scalable).

Brainstorm parents

  • Constraint-Satisfaction Seating Puzzle (unique-solution stratum) — the brainstorm seed folded in as the CSP-flavor stratum of the merged candidate.
  • Merged entry (origin: both) — merges the seating puzzle with the RuleTaker/ProofWriter/FLD/PrOntoQA literature cluster into one candidate.

Dead branches: dropped near-relatives

  • FOLIO as a scalable source — downgraded to style reference only: human-written, ~1.4k items, not synthetically regenerable, so it cannot anchor a seeded infinite pool.
  • Verbatim reuse of published RuleTaker/ProofWriter/PrOntoQA sets — refused because the released theories are contaminated; the adaptation must regenerate with fresh symbols.
  • Full proof-chain generation as output — dropped from the design; keep the label only to bound output tokens and preserve exact-match scoring.
  • (Not this candidate) the LLMStructBench unverifiable-provenance flag — belongs to c12's extraction cluster, not c18; no laundering needed here.

Related active research (2025-2026)

  • JustLogic: A Comprehensive Benchmark for Evaluating Deductive Reasoning in LLMs (Chen, Zhang & Tao, arXiv 2025) — a synthetic, prior-knowledge-independent deductive benchmark with fine-grained error analysis over reasoning depth and argument form; a direct 2025 successor to the c18 family.
  • Multi-LogiEval (Patel et al., EMNLP 2024 Main) — a depth-1-to-5 multi-step logic benchmark over 60+ inference-rule combinations (propositional/FOL/non-monotonic); a relevant depth-ladder reference, though 2024 rather than strictly 2025-2026.
  • c18 related work — deep-read dossiers + capability/positioning synthesis, 2026-07-22.
  • c18 baseline spec — draft baseline experiment spec — headroom before complexity.

Supports
  • Scored 27 with a tie-break of 2; the landscape names it "the cleanest incremental ladder" via proof-depth bins, and the proof engine names the failed step for reflection.
Against
  • It sat just below the 28-tier finalist cut, held back by the heavier reasoning load per instance.

Optimization-path skeptic

Supports
  • Fresh symbols per run and depth binning make the decisive knowledge (chaining depth, negation-as-failure, open-world Unknown) discoverable from demos rather than one-shot guessable, with a designer-known ceiling.
Against
  • This candidate never received a dedicated round-1 skeptic pass — the adversarial lenses were run on the seven 28-tier finalists, not on 27-tier tasks, so no skeptic verdict below the finalist tier examined it.

Measurement skeptic

Supports
  • The label is a single T/F/Unknown token under exact match, and the run verified 3/3 hand-traced labels correct with proof chains derivable purely from in-instance rules — no hidden knowledge.
Against
  • The emitted label is definitional (a 50%-probability negation flag), not an independent prover verdict, so a generation soundness bug would not be caught by the label alone; and the CWA-vs-Unknown boundary can be ambiguous without a precise reference prompt.

Implementation lens (round 2)

Supports
  • The codebase is run-verified deterministic (byte-identical at seed 12345, varying at 999) and the adaptation is ~100 lines of external glue with no required repo edits.
Against
  • Design breadth costs it: a second constraint-puzzle stratum plus CWA/Unknown pinning keep the harness-shape and gate scores low, so the total lands mid-pack rather than at the top.

Research currency

Supports
  • The deductive-reasoning line is alive in 2025-2026 (JustLogic 2025, SATBench EMNLP 2025, FLD* NeurIPS 2024), and prontoqa is actively cited in current reasoning-eval papers.
Against

Implementation: steps & risks

Subprocess generation per stratum

Call run_experiment.py --model-name json per depth stratum, looping --min-hops/--max-hops for the D0-D5 ladder, --ontology fictional for nonce vocabulary, and --distractors plus deduction rule as difficulty knobs; run from the repo root so the relative bad_patterns.txt import resolves (~40 lines of driver plus an arg table).

Parse the JSON into the instance schema

Read example{i}.test_example.{question,query,answer}, treating answer as the 0/1 exact-match gold and dropping the optional chain_of_thought (~30 lines).

Single scored LLM call

Concatenate question plus query, ask for True/False/Unknown, exact-match against answer (~20 lines, oracle-independent of the model under test).

Add the independent oracle and the CSP stratum

Write a ~30-50-line forward-chaining fixpoint check so the label is re-derived rather than trusted as the definitional negation flag, and add the unique-solution seating stratum for the CSP flavor.

Risks

  • The T/F/Unknown label is definitional (a negation flag), not an independent prover verdict, so a generation bug is invisible in the label alone — mitigate with the forward-chaining fixpoint check.
  • CWA-versus-Unknown semantics needs a precise reference prompt (2-3 pre-freeze gates) or the Unknown/False boundary is ambiguous.
  • --deduction-rule Composed with postorder ordering crashes (verified, 0-byte output) — use --ordering random; and small concept pools with high hop counts loop on retry-by-rejection, so size the vocab to the requested depth.

Repo review & run verification

  • Codebase: asaparov/prontoqa (Apache-2.0, maintained, actively cited) — deep-read notes at repos/asaparov-prontoqa.md.
  • Run verification: verified --seed threads into both random and np.random; two runs at seed 12345 were byte-identical, seed 999 differed, and 3/3 hand-traced labels plus proof chains were correct and derivable purely from in-instance rules. Only new caveat: must run from repo root for bad_patterns.txt.
