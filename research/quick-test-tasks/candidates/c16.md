**Status:** RANKED #18 of 20 (round-2 rescore)

**You know this as:** the Wason conditional-reasoning paradigm (if-P-then-Q rule checking) — reframed from the four-card selection puzzle into a single-call OK/VIOLATION (or which-rule-violated) compliance classifier over attribute records.

## Verdict

**27-tier candidate, dropped to #18 after the implementation-aware rescore because no reusable codebase exists.** On the round-1 design this candidate carries a genuinely rare property: it is the one task in the pool that retains real model-side residual headroom, because models systematically err on vacuous-truth handling and rule directionality even when the conditional rules are fully stated, so a "perfect" instruction does not instantly reach the ceiling the way it does for the execution-heavy tasks. The scoring surface is clean — a record of four to six visible attributes is checked against four to five conjoined implication rules, and the model emits OK/VIOLATION or the violated-rule letter for exact match — and attribute-combinatorial seeded generation is contamination-proof. What sank it in round 2 is the same fact that sank c12, c13, and c14: discovery found no plausibly reusable generator, so the double-weighted reuse axis scored zero and every implementation hour would be from scratch. The residual-headroom virtue is also its calibration liability, since the binary/MC output and the vacuous-truth subtlety make the floor hard to pin precisely without empirical gating.

## What it proposes

### A conditional-compliance classifier where the label names whether an implication rule was broken

A seeded generator emits **records** of four to six visible attributes (shape, edge, color, count, and distractor fields) and checks each record against four to five conjoined **implication rules** of the form "if shape = circle then edge = solid." A record is **VIOLATION** if and only if some rule's antecedent holds while its consequent fails; cases where the antecedent never fires are **vacuously OK**. The model outputs either a bare **OK/VIOLATION** boolean or, in the richer variant, the **violated-rule letter**, which escapes the fifty-percent guess floor and turns each failure into a diagnostic pointing at one rule. Ground truth is exact match against a trivial boolean evaluation of the rule set. The latent rules that decompose difficulty are each individual implication, correct **vacuous-truth** handling, **directionality** (the rules are one-way implications, not biconditionals), and the presence of **distractor attributes** that no rule references. The distinctive property is that models keep erring on vacuous and directional cases even under a fully-stated ruleset, so the ceiling is not instantly reachable — an unusual and desirable model-side residual for a prompt-optimization ladder.

### Example instance (illustrative)

*Illustrative, built faithfully from the merged summary. Seeded conditions: this run draws three implication rules over shape/edge/color, includes one vacuously-satisfied antecedent and one distractor attribute (size), and uses the which-rule-violated output to avoid the bare-boolean guess floor.*

```
Rules (this run):
  R1 : if shape = circle  then edge  = solid
  R2 : if color = red     then shape = square
  R3 : if edge  = dashed  then count > 2

Record:  shape = triangle,  edge = dashed,  color = red,  count = 4,  size = large

Question: does this record comply with all rules? If not, which one is violated?
  Answer OK, or the letter of the single violated rule.

Gold: R2  (color=red fires R2's antecedent but shape=triangle, not square → violation.
           R1 is vacuous: shape is not a circle, so R1 never fires. R3 holds: edge=dashed
           and count=4 > 2. size is a distractor no rule mentions.)

Naive-prompt miss: a solver that reads implications as biconditionals flags R1
("edge is dashed, not solid") or treats the vacuous R1 antecedent as a failure,
and answers R1 — it never internalized directionality or vacuous truth, the two
error classes that survive a perfect instruction.
```

## Perspectives

### Round-1 rubric fit
- **Supports:** Scored 27 with a rare bonus among all candidates — genuine model-side residual headroom (vacuous-truth and directionality errors survive a fully-stated instruction), plus bounded exact-match output and independent per-implication rule decomposition.
- **Against:** The bare-boolean OK/VIOLATION output gives a fifty-percent guess floor, so it needs the which-rule-violated MC variant and balanced vacuous/violating cases to keep guessing penalized and the ladder meaningful.

### Optimization-path skeptic
- **Supports:** Attribute-combinatorial seeded generation is contamination-proof, and the decisive knowledge (which implications hold, plus vacuous/directional handling) is not one-shot enumerable in a way COPRO's score-only rephrasing can trivially recover.
- **Against:** No skeptic pass examined c16 below the 28-tier finalist cut; it sits in the 27 tier where the optimization-path lens did not run a dedicated refutation, so this reading is inferred from the cluster analysis rather than a recorded verdict.

### Measurement skeptic
- **Supports:** The exact-match boolean/MC surface makes effect sizes clean to read, and the residual-headroom property means a well-tuned prompt still leaves a measurable gap rather than saturating to ceiling.
- **Against:** No dedicated measurement-skeptic verdict exists for c16 (it was below the finalist tier that received adversarial passes); the merge itself flags that the binary/MC output and vacuous-truth subtlety make the floor "harder to pin precisely," the honest open measurement concern.

## Implementation: steps & risks

1. **Author the record + rule sampler from scratch.** There is no donor repo, so write `c16_gen.py` with a threaded `random.Random(seed)` that draws four to six attributes over small closed vocabularies and four to five conjoined implication rules over those attributes; this is the entire generator, built new.
2. **Write the trivial boolean oracle.** For each record, evaluate every rule as a one-way implication (antecedent-holds-and-consequent-fails == violation) and take the vacuous case as OK; this second path is independent of any prompt and hand-checkable in under a minute, securing the oracle-independence score.
3. **Choose the output surface and balance cases.** Prefer the which-rule-violated MC label over bare OK/VIOLATION to escape the fifty-percent guess floor, and balance vacuous, violating, and clean cases across strata so no single class dominates.
4. **Seed the residual-headroom strata.** Deliberately include vacuous-antecedent records and directional (non-biconditional) traps so the vacuous-truth and directionality error classes are exercised, since those are the model-side residuals the task exists to measure.
5. **Calibrate the floor and ceiling empirically.** Because the floor is hard to pin, run pre-freeze gates to confirm a designer ceiling prompt reaches near-100% while a naive prompt leaves measurable residual, and audit for dead strata where every instance is trivially OK.

### Risks
- No reusable codebase exists, so the whole generator, oracle, and calibration harness are from-scratch work — the decisive reason for the round-2 drop and the "do not invest" listing.
- The floor is hard to pin: the bare-boolean variant guesses at fifty percent and the vacuous-truth subtlety can make some strata unexpectedly easy or unexpectedly adversarial, so calibration gates are mandatory rather than optional.
- If the rules read as biconditionals to the model, the directionality residual can dominate and skew the ladder; balancing directional and vacuous cases is required to keep the strata legible.
