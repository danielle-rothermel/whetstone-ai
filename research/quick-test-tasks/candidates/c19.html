c19Grid-World State Prediction

ADOPT
You know this as: grid-world state tracking — the bAbI/BabyAI "predict the final state after a move sequence" task, reframed to one derived fact.

Adopt — ranked first after the implementation-aware rescore. c19 wins on the single fact that dominated round 2: it is the only candidate that pairs a genuinely reusable, run-verified codebase with skeptic debt that is verification-only rather than structural. Farama-Foundation's Minigrid is Apache-2.0, actively maintained, and — confirmed by a fresh run — properly seed-threaded, so the same seed reproduces byte-identical grids and different seeds vary. The entire adaptation is roughly a hundred-and-a-half lines of configuration, an independent derived-fact oracle, and strict all-or-nothing scoring glue that lives outside the repo; nothing inside minigrid/ is edited. Round 1 scored the design in the strong middle of the pool and flagged only one soft spot — grid navigation is a familiar enough skill that a naive prompt might not floor low — which the invented-command and wrap-versus-clamp convention strata are designed to counter. Because those nonstandard dynamics are net-new logic we write on top of Minigrid rather than something the repo supplies, the build is real but bounded, and the standing reflects that: a mid-pack round-1 task that rose two tiers to the top on implementation strength.

What it proposes

Predict one derived fact from a small seeded grid, exact-match

A seeded generator builds a small ASCII grid (roughly six-by-six) with walls, a robot pose, and a short command string that mixes ordinary moves with two or three invented commands (for example J = jump-two, W = wrap). The model predicts exactly one derived fact — a final coordinate, a heading, or a carrying-flag — never a path, which removes the many-valid-answers ambiguity that plagues plan-generation grid tasks. Output is a single token (3,5 or E), scored by exact match against a tiny independent simulator. The latent rules that vary across strata are relative-turn semantics, wall-blocking versus edge wrap or clamp, the invented-command semantics (including jump-through-wall behavior), and the origin/indexing convention — varied so that no single fixed convention statement can cover the whole pool. Randomized grids are contamination-proof, and an off-by-one-at-a-wall failure is cleanly diagnostic of which rule the prompt missed.

Academic anchor & lineage

Literature parents

  • GridRoute (Li et al., arXiv 2025) — cardinal-movement grid route-planning benchmark; c19 flips its plan-generation into single-token state prediction.
  • GRASP (Tang & Kejriwal, arXiv 2024) — sixteen-thousand grid-based commonsense spatial-reasoning environments; the direct grid-world antecedent.
  • LLM-BabyBench (Choukrani et al., arXiv 2025) — textual BabyAI grid world; its Predict split (final state after an action sequence) is the c19 template.
  • bAbI tasks (Weston et al., arXiv 2015) — classic synthetic entity/world-state tracking QA; c19 regenerates its state-tracking flavor over grids.
  • Recurrent Entity Networks / EntNet (Henaff et al., ICLR 2017) — world-state tracking model that solves bAbI; cited as evidence for state-tracking difficulty and ceiling.
  • Entity Tracking in Language Models (Kim & Schuster, ACL 2023) — probes LM tracking of an entity's final state after operations; motivates the single-derived-fact framing.

Brainstorm parents

  • Grid Robot Deterministic Navigator (brainstorm) — the seed idea: a deterministic robot walking a small grid under stated movement rules.
  • c19 in candidates-merged — origin "both"; the merge fused the brainstorm navigator with the four literature grid/state-tracking lines above into a single state-prediction task.

Dead branches: dropped near-relatives

  • D5 · Grounded grid compositional (gSCAN / ReaSCAN) — grid rendering plus action-sequence outputs are the most expensive things in the pool to bound, and the reviewer judged it "not worth the adaptation cost"; grid coverage was assigned to c19, compositional splits to c23.
  • D4 · CRUXEval-style code-output prediction — the published set is heavily contaminated and regeneration needs a code DSL with unconstrained return types that fight exact-match bounding; a cousin execution-simulation branch killed on the same bounding grounds — execution flavor went to c05, rule-induction to c23.

Related active research (2025-2026)

  • MazeEval (Einarsson et al., arXiv 2025) — coordinate-based maze navigation with distance-to-wall feedback; models fail beyond nine-by-nine via looping, directly probing the grid state-prediction floor c19 depends on.
  • Do LLMs Build Spatial World Models? Grid-World Maze Tasks (Li et al., arXiv 2026) — synthetic five-by-five to seven-by-seven grid mazes where accuracy swings from sixteen to eighty-six percent by representation and prompt format, validating c19's format-sensitivity and headroom claims.
  • c19 related work — deep-read dossiers + capability/positioning synthesis, 2026-07-22.
  • c19 baseline spec — draft baseline experiment spec — headroom before complexity.

Perspectives

Round-1 rubric fit

Supports
  • Asking for one derived fact rather than a path removes the many-valid-answers ambiguity and yields a bounded single-token exact-match target; randomized grids are contamination-proof and off-by-one-at-a-wall failures are cleanly diagnostic.
Against
  • Grid navigation is a familiar skill, so the naive-prompt floor risks sitting too high unless the invented-command and wrap-versus-clamp strata push it down — the one soft spot round 1 named.

Optimization-path skeptic

Supports
  • The latent rules (turn semantics, wrap versus clamp, invented-command behavior, indexing) are varied so no single fixed convention statement covers the pool, keeping decisive knowledge outside score-only instruction rephrasing.
Against
  • No dedicated round-1 skeptic pass examined c19 below the 28-tier — recommendations.md analyzes only the five finalists, so this lens on c19 is inferred from the merged summary, not an adversarial verdict.

Measurement skeptic

Supports
  • Ground truth is read directly from the constructed grid via an independent object-model walk, not re-derived from the sampler, and three of three run-verified hand-checks matched — the oracle is independent and human-verifiable in under a minute.
Against
  • Likewise never given a formal measurement-skeptic pass in round 1; the ceiling and per-stratum floors remain unmeasured pre-freeze gates rather than confirmed properties.

Implementation lens (round 2)

Supports
  • The only run-verified I1=3 codebase in the pool with verification-only skeptic debt — properly threaded seeding, byte-identical reproduction, roughly a hundred-and-a-half lines of glue, and zero edits inside minigrid/.
Against
  • Minigrid supplies generation, serialization, and determinism but not the nonstandard dynamics — the invented-command and wrap-versus-clamp convention layer is net-new simulation logic we write, which is where the real build effort sits.

Research currency

Supports
  • Grid-world state prediction is live in 2025-2026 — MazeEval and the 2026 spatial-world-models maze study both probe exactly this floor and its format sensitivity, and Minigrid is actively used in current work.
Against
  • Grid navigation is a heavily trodden benchmark family, so the contribution is a careful reframing rather than a new problem, and spatial reasoning carries known modality confounds that must be controlled by keeping grids small and ASCII.

Implementation: steps & risks

Day 1-2 — build the harness

Vendor nothing; add Minigrid as a dependency. Write the roughly-hundred-and-a-half-line glue package: a seeded instance generator over stochastic-layout envs (Fetch, Crossing, FourRooms, Empty-Random), text render via pprint_grid, the invented-command and convention layer (wrap versus clamp, indexing, two or three invented commands) as a tiny stepper over grid.encode state, an independent derived-fact oracle, and a strict all-or-nothing scorer.

Day 2-3 — freeze gates

Run a reference "state-all-rules" ceiling prompt at temperature zero; measure per-stratum floors; run a red-team check that no single fixed convention statement covers the pool. Pin seeds and a manifest.

Wire strata and scoring

Vary env id and grid size as strata (seed = instance index), optionally remap COLOR_NAMES / IDX_TO_OBJECT to nonce tokens as pure text post-processing, then a single LLM call compared against the oracle for exact match.

Risks

  • The invented-command and wrap-versus-clamp convention strata that defeat one-shot guessing are our net-new simulation-layer logic; Minigrid does not supply nonstandard dynamics, so a bug there is a bug in the part that makes the task hard.
  • Fixed-layout envs do not vary by seed (Empty-8x8 is identical for seeds 7 and 8) — strata must use the stochastic-layout envs, and the off-path unseeded spots (wrappers.py:800, the WFC subpackage) must be avoided.
  • Roughly two to three pre-freeze gates remain (measured ceiling, per-stratum floors, single-fixed-convention red-team) — real calibration, not zero.
  • Minor license-metadata mismatch: the LICENSE file says Apache-2.0 while pyproject says MIT; cite the LICENSE file.

Repo review & run verification

  • Codebase: Farama-Foundation/Minigrid (Apache-2.0, v3.1.0) — deep-read notes at repos/farama-foundation-minigrid.md.
  • Run verification: verified seed 42 byte-identical across runs on five envs; seed 42 vs 43 differs; three of three ground-truth spot checks matched an independent object-model walk. Seeding threaded through reset(seed) → self.np_random with no module-global seeding.
