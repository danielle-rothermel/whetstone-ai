Why does it rank fourth, not higher?

Ranked fourth of twenty in the round-2 rescore, up from the round-1 27-tier into the top five. It rose because its codebase is real and run-verified: reasoning-gym is consumed as a clean dependency with no repo edits, its per-item Random(seed+idx) plumbing is threaded and byte-identical under re-run, and its sorting oracle is Python sorted() — trivially independent.

It sits behind the podium because the differentiating task logic is entirely net-new: the randomized composite tie-breaker keys that give each rule level a distinct diff signature are not in the shipped generator, so a fresh subclass of roughly 100–150 lines must be written, and the base class awards substring partial credit that a strict 0/1 override must replace. A single shared reference sorter couples the strata to re-certification, which is what holds its rule-independence cell below the top.

What does the task ask a model to do?

Sort short token lists by a per-batch invented composite key, exact-match the canonical line

Each instance is three to six tokens or (letter, number) pairs — or, in one stratum, three to four small permutations in cycle notation with a stated composition-direction convention. A nonstandard composite key governs the ordering: sort by the second letter, break ties by length descending, rerank vowels after consonants, apply a per-element transform (double if even, triple if odd), and emit canonical cycle form with the smallest element leading.

The model returns one comma-joined line or canonical string, scored by exact match against a reference sorter. Each latent rule — primary key, each tie-breaker, the rerank rule, the element transform, the separator convention — is visible as a distinct diff signature, so a partial prompt fails on exactly the rung it missed. Keys and directions are randomized per batch so the prompt must convey them rather than let the model guess from priors.

Where does the task come from?

Literature parents

  • BIG-Bench Hard (Suzgun et al., ACL Findings 2023; arXiv 2022) — supplies the word_sorting task, its exact-match short-output format, and the large prompt-sensitive headroom this candidate inherits. Note the arXiv-vs-venue split: first posted 2022, published ACL Findings 2023.
  • BIG-bench (Srivastava et al., TMLR 2022) — the word_sorting task actually originates here, not in the BBH paper; BBH merely selects it. Underlying substrate for the sorting instance format.
  • OPRO (Yang et al., ICLR 2024) — named optimizer substrate; reports up to +50% on BBH tasks, the strongest headroom evidence for this sorting format.
  • EvoPrompt (Guo et al., ICLR 2024) — named optimizer substrate reporting BBH gains, co-anchoring the prompt-sensitive headroom claim for word_sorting.
  • Promptbreeder (Fernando et al., arXiv 2023) — evolutionary optimizer in the same BBH/word_sorting lineage, producing BBH exact-match gains.
Corpus correction: c24's "Merged from (literature)" line names only the bare "BBH word_sorting / symbolic-subset generators (OPRO/EvoPrompt substrate)" reference; the individual papers above are named only in research-prompt-opt-literature.md and are surfaced explicitly here.

Brainstorm parents

  • Invented-rule sorting of symbolic tokens — the seed idea: order tokens under a rule the model cannot guess.

  • c24 merged entry (origin: both) — folds the brainstorm sorters together with the BBH word_sorting generator substrate.

Dead branches: dropped near-relatives

  • Inverted Multiple-Choice convention (dropped cluster D1) — both sources flag HIGH one-shot risk; a single sentence ("answer the alphabetically-first incorrect option") closes the gap, failing the anti-one-shot criterion; remapped-label ground is covered by c15.
  • Interval-Overlap Field Selection (dropped cluster D2) — only two or three boundary/tie conventions, enumerable in one prompt; the boundary-convention flavor is absorbed by the c12/c13 strata.
  • Char-level string-task cluster (dropped cluster D9; StringLLM, CUTE, letter-counting) — not a task — kept only as the red-flag caveat that char-level counting is tokenization-bound and temp-0 noisy, which is why this candidate stays at the token level, not the character level.

Is the underlying difficulty still research-current?

  • Premise Order Matters in Reasoning with Large Language Models (Chen et al., ICML 2024) — best-known evidence that reordering under a fixed convention swings LLM accuracy by more than thirty points; directly motivates the ordering-under-invented-conventions difficulty this candidate targets.
  • Order Doesn't Matter, But Reasoning Does: Training LLMs with Order-Centric Augmentation (He et al., EMNLP 2025) — 2025 work on LLM brittleness to premise and condition ordering under logically-equivalent reorderings, keeping the ordering-under-invented-conventions premise research-current.

What has to be built, and what could break?

1 · Consume reasoning-gym as a dependency

Vendor or install open-thought/reasoning-gym and reuse its ProceduralDataset infra and per-item Random(seed+idx) pattern; change zero repo lines.

2 · Write the composite-key sorting subclass

New ProceduralDataset subclass, roughly 100–150 lines: generate records with several invented fields, sample a randomized composite key ordering (primary plus tie-breakers) per instance as the latent rule, and compute the oracle with Python sorted(key=...).

3 · Override scoring to strict 0/1

Replace the base class's substring partial-credit score_answer with a strict exact-match 0/1 override, so a prompt that misses one rule level scores zero on that instance.

4 · Swap in nonce vocabulary and add glue

Bypass the bundled real-text Gutenberg corpus (in_the_year_2889.txt) with nonce vocab to keep instances contamination-proof; add roughly 30–50 lines of LLM glue for the single call and 0/1 exact-match scoring.

Risks

  • The randomized composite tie-breaker keys are net-new task logic, not a config flag — the shipped generator sorts by a single ASCII key with no per-tie-breaker diff signatures
  • A shared reference sorter couples every stratum to re-certification: a change to the oracle re-validates all key levels at once (this is the I5=1 cause)
  • If the strict override is forgotten, the base class's substring partial credit silently breaks the intended 0/1 exactness
  • Off-path pool_matrix.py uses module-global np.random.seed; do not co-import it naively into a shared process, or determinism can be corrupted

Repo review & run verification

  • Codebase: open-thought/reasoning-gym (Apache-2.0) — deep-read notes at repos/open-thought-reasoning-gym.md.
  • Run verification: verified — ran in a fresh venv; same-seed byte-identical, different-seed varies, and all six hand-checked word_sorting instances matched an independent sorted() oracle. The invented composite-key semantics themselves are never run-verified — they are the net-new code still to be written.
