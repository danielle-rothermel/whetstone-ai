## What it proposes

### The task: execute a seeded abstract rewrite system to its unique normal form

The generator emits an instance made of **3–6 productions** over **invented symbols**
plus a **start string of 8–14 symbols**. The system is constrained to be **confluent and
terminating** (or a **fixed leftmost-innermost strategy** is imposed) so the **normal
form** is unique and program-checkable. The model must apply the productions,
**iterating to fixpoint** (not a single pass), and return the short normal-form string —
or, in the **L-system parallel-rewriting stratum**, a **bounded derived feature** (final
length, or the symbol at position *k*) so the answer stays short even though a D0L
derivation blows up exponentially. Scoring is exact match against a **reference
rewriter** run to fixpoint. Latent rules decompose cleanly: each individual production,
the redex-selection strategy, iterate-to-fixpoint, empty-production deletion, guard
conditions, and parallel-vs-sequential application. Failures localize to a single
un-applied production or a halted iteration — an ideal reflection signal.

### Example instance (illustrative — constructed from the merged summary)

Seeded conditions: a nonce alphabet, a confluent+terminating rule table, a start string
in the mid-length band; the answer is the irreducible string.

```
Rewrite rules (apply anywhere, repeat until nothing fires):
  ØÞ → Þ
  ÞØ → Þ
  ØØ → Ø
  ÞÞ → Þ

Start:  Ð Ø Þ Ø Þ Ø Þ Ø Þ Ð

Question: give the normal form (the string when no rule can fire).

Gold: ÐÞÐ      (reference rewriter, run to fixpoint; confluent, so order-independent)

Naive miss: a one-pass left-to-right scan applies each rule once and stops, missing
that new redexes appear after each rewrite (each collapse creates a fresh ØÞ/ÞØ pair
against its neighbour). Only iterate-to-fixpoint reaches ÐÞÐ.
```

The abstract-symbol framing is the load-bearing move: no pretraining prior tells the
model what `ØÞ → Þ` means, so the demo and the rule table must carry the whole skill.

## Perspectives

### Round-1 rubric fit
- **Supports:** Arguably the purest rubric fit — abstract symbols, exact program-checkable oracle, per-production decomposition, single call, bounded output; failures localize to one misapplied production (ideal for reflection).
- **Against:** Scored 26 with tie-breaker 1, the flag being that it is *potentially too hard* — the difficulty may sit above the band where an incremental ladder is observable.

### Optimization-path skeptic
- **Supports:** The strongest anti-one-shot design in the pool — abstract productions over invented symbols have nothing to guess or recall, so the demo/rule table is unavoidably load-bearing.
- **Against:** No dedicated skeptic pass ran on c04 (it sat below the 28-tier where the adversarial lenses were applied), so the optimization-path story is asserted from the design, not stress-tested.

### Measurement skeptic
- **Supports:** A confluent+terminating system (Newman's lemma) or a fixed reduction strategy makes the normal form provably unique and exactly checkable — no ambiguous labels.
- **Against:** Character-level normal forms invite temp-0 token noise on longer outputs (landscape.md L129/L205–207 carries this "char-arithmetic temp-0 noise" repair flag), and determinism must be actively enforced or the label is ambiguous.

### Implementation lens (round 2)
- **Supports:** Ranked #7 of 20, it rides the run-verified reasoning-gym base: a clean, independent term-rewriting generator and oracle already exists there (`ab.py`), on a re-seedable framework whose ProceduralDataset / score_answer / register_dataset contract matches every c04 requirement — earning I1 = 3.
- **Against:** The full c04 spec (invented vocabularies + latent-rule strata + MC variant) is *not native*: rules and symbols are fixed in `ab.py`, so a ~150-LOC clone-and-generalize (Option B) is required, and the adapted design's many interacting parts drive I4 to 0 and I3/I5/I6 to 1. The candidate-specific rewrite-system logic is authored on top of the run-verified base rather than run-verified itself.

### Research currency
- **Supports:** Grounded in an actively-maintained NeurIPS 2025 Spotlight substrate plus 2024–2025 arXiv work on confluence-strategy invention and L-system benchmark generation.
- **Against:** The core theory (Terese, Baader & Nipkow) is textbook-old, and the recent L-system paper targets compiler benchmarks rather than QA, so currency is partly borrowed from the substrate rather than the task itself.

## Implementation: steps & risks

*(Implementation scoring was produced by the same pipeline as every other candidate — web discovery + desk scoring with the same rubric and exercises + a deep active-path code read written to repos/. Run-verification: open-thought/reasoning-gym was run-verified during round 2 under c24 (same-seed byte-identical, 6/6 independent ground-truth checks), so repo-level verification carries over; the c04-specific rewrite-system logic is authored on top of that reused, verified infrastructure.)*

1. **Adopt reasoning-gym as the generator+oracle substrate.** Reuse `ProceduralDataset`, per-instance `Random(seed+idx)` seeding, `register_dataset`, and the exact-match `score_answer` (already strict 0/1).
2. **Clone-and-generalize `ab.py` into a seeded rewrite generator (Option B, ~150 LOC).** Sample a nonce alphabet, a rule table, and a start term from the seed; generalize `compute_steps` from four hard-coded productions to a rule-table lookup; render the prompt from the sampled table.
3. **Enforce determinism before freeze.** Certify confluence + termination (critical-pair check / Newman's lemma) or impose a fixed leftmost-innermost strategy so every instance has a unique normal form.
4. **Add latent-rule strata and the L-system stratum.** Copy the curriculum pattern for rule-count / alphabet-size levels; add a capped-iteration D0L stratum that asks for a bounded derived feature; add the optional MC-over-candidate-normal-forms variant.
5. **Calibrate difficulty and pin the noise floor.** Tune term length and rule count so a default prompt lands mid-range; measure temp-0 character-level noise on longer normal forms and, if needed, shorten outputs or move to the bounded-feature/MC framing.

**Risks**

- *Add-one-rule blast radius is global.* A new production can break confluence/termination via critical-pair overlaps, so the confluence check must re-run on the whole system and every instance whose normal form changes must be re-verified and regenerated — the rule is not rule-local (I5 = 1).
- *Four-plus pre-freeze gates.* Determinism enforcement, "potentially too hard" difficulty calibration, the char-arithmetic temp-0 noise floor on longer normal forms, and the L-system iteration cap all must clear before freeze (I4 = 0).
- *Two open repair flags, no skeptic pass.* landscape.md L129/L205–207 records "potentially too hard" and "char-arithmetic temp-0 noise"; both are repairable but non-empty, and no adversarial lens examined c04 below the 28-tier (I6 = 1).

**Repo review & run verification**

- **Codebase:** [open-thought/reasoning-gym](https://github.com/open-thought/reasoning-gym) (Apache-2.0) — deep-read notes at [../repos/open-thought-reasoning-gym.md](../repos/open-thought-reasoning-gym.md). Ships `ab.py` (A::B term-rewriting-to-normal-form, deterministic `compute_steps` oracle, length curriculum) and `string_synthesis.py` (second rewrite-to-fixpoint generator).
- **Run verification:** the reasoning-gym base was **run-verified during round 2** (under c24: same-seed byte-identical, 6/6 independent ground-truth checks), so repo-level verification carries over — same-seed reproducibility, independent oracle, and 0/1 exact match all reproduced empirically in the deep-read. The candidate-specific Option-B rewrite generator is authored on top of that verified base and is not itself separately run-verified. Distant references amarmaduke/reducto (MIT) and gregcaban/rkt-term-rewriting (MIT) supply reduction semantics only, with no seeding, strata, or exact-match harness — not config-plus-glue reuse.
