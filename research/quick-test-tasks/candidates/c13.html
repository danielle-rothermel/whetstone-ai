Precedence-Ordered Decision-List Classification ranks #17 — good task, no code

RANKED #17

A first-match ordered rule list (CORELS-style) with overrides acts as a latent triage-label oracle.

whetstone-ai · quick-test-tasks · round-2 implementation-aware rescore

This is the archetypal independent-rules-with-override task, and on the round-1 rubric it scored well — the first-match ordering gives genuine rule interaction that resists one-shot specification, each rule owns a case slice so partial rule knowledge earns partial pool score, and a feature-combinatorial seeded pool is contamination-proof by construction. What sank it in round 2 is entirely on the implementation axis: no plausibly reusable codebase exists, so every hour of the build is from-scratch. Its desk profile is otherwise fine — a second oracle is trivial (exhaustively close the feature grid), calibration is light, and round 1 left no repair flags — but the double-weighted reuse dimension scoring zero drops it into the bottom tier alongside the other no-repo tasks. It is a good task with no code to stand on.

What does it actually ask the model to do?

What it proposes

Label one record by executing a latent first-match rule list with overrides

The generator samples a short ordered decision list (first-match semantics) over four to six categorical or numeric features, plus a set of override rules — security=true forces P0, a senior veto, a regional-cutoff shift, a paying-customer never-SUPPRESS floor — arranged in an explicit precedence chain. Each instance is one record (or vote set, or event); the model emits one label from a roughly five-way set; scoring is exact match against executing the list top-to-bottom and returning the first rule that fires. Because each rule governs a distinct case slice, partial rule coverage yields partial pool score and failures are diagnostic: "all security=true cases wrong" localizes precisely to the override that was missed. Boolean-concept complexity — rule count, feature arity, override-chain depth — is the principled difficulty dial.

Example instance

Illustrative — constructed from the merged summary. A five-way label set relabeled with opaque symbols to defeat priors; a first-match list where a security override precedes a region downgrade which precedes a paying-customer floor.

Rules (latent, in precedence order):
  R1: if security = true            -> P0
  R2: if severity >= 4              -> P1
  R3: if region = APAC              -> downgrade one level
  R4: if plan = paying              -> never SUPPRESS
  R5: otherwise                     -> P3

Record:  severity=4, security=false, region=APAC, plan=free
Gold:    P2      (R2 fires -> P1, then R3 downgrades -> P2)

Naive-prompt miss: a model told only "classify the incident severity"
returns P1 (applies R2, ignores the latent APAC downgrade in R3).

Where does the design come from?

Academic anchor & lineage

Literature parents

  • Learning Certifiably Optimal Rule Lists for Categorical Data (Angelino et al., KDD 2017 / JMLR 2018) — CORELS; the best-known source for first-match ordered rule lists, which is the precedence/override oracle at this candidate's core.
  • Rule Generation for Classification: Scalability, Interpretability, and Fairness (Röber, Lumadjeng, Akyüz & Birbil, arXiv 2021) — MIP / column-generation rule sets, representative of the "optimal rule sets/lists" line. The corpus cited this arXiv id as "Yu et al. (MaxSAT), 2020" — surname, title, method, and year are all wrong.
  • RuleArena: A Benchmark for Rule-Guided Reasoning with LLMs (Zhou et al., ACL 2025) — real-world rule-application benchmark; supplies the "synthetic rule engine" framing for independent-latent-rule decomposition.
  • Minimization of Boolean Complexity in In-Context Concept Learning (Wang, McCoy & Steinert-Threlkeld, arXiv 2024) — in-context Boolean concept learning; supplies the principled Boolean-complexity difficulty dial.
  • LLMTabBench: Evaluating LLMs on Binary Tabular Classification From Zero to Few Shots (Grushina et al., arXiv 2026) — synthetic tabular data plus generator-produced threshold decision rules; the corpus's proposed drop-in generator. Title and authors confirmed via arXiv, but the released artifact/license was not independently corroborated (v1 2026-05-23).

Brainstorm parents

  • Priority Override Router — the "independent-rules with override" sketch (Traffic-Light Priority Classifier, Priority Vote Aggregator, Temporal-Window Event Labeler, Priority-Conflict Record Reconciler, Orthogonal-Feature Labeling) that this candidate crystallizes.
  • Merged entry c13 — origin "both" (brainstorm + literature); folded the vote-aggregation and event-window flavors into the single first-match-list core.

Dead branches: dropped near-relatives

  • D2 Interval-Overlap Field Selection — only two or three boundary/tie conventions, enumerable in one prompt; the boundary-convention flavor was absorbed into c12/c13 strata rather than kept as its own task.
  • D7 Contaminated classification templates (Ethos, Liar, SST-2/Subj/TREC/AG-News suite) — single latent construct, contaminated, subjective labels; only their cheap single-call MC output format was inherited by c13/c15's synthetic reframes.

Does the case hold up from every angle?

Perspectives

Round-1 rubric fit

Supports
  • Excellent on the decomposition criteria — synthetic and contamination-proof, a known 100% ceiling (state every rule), cleanly decomposable independent rules, first-match precedence for genuine interaction, and diagnostic "which rule fired" failures.
Against
  • Its rule families (security override, region downgrade, paying floor) are less contamination-novel than the nonce-graph or opaque-codebook designs, which is exactly why it was edged out of the 28-tier finalist cut.

Optimization-path skeptic

Supports
  • The first-match ordering makes rule interactions non-trivial and not one-shot guessable, and relabeling the classes with opaque symbols defeats any residual triage prior, so the gap is a real learnable one across prompts.
Against
  • Over-simple feature encodings can be one-shot solved by a competent prompt engineer; difficulty must be actively controlled through rule count, feature arity, and override-chain depth or criterion 4 collapses.

Measurement skeptic

Supports
  • Boolean-concept complexity is a principled difficulty dial, and each rule owning a distinct case slice means pool accuracy climbs rule-by-rule with an inspectable ladder.
Against
  • No dedicated skeptic pass examined this candidate below the 28-tier finalists — the measurement critique here is inferred from the domain deep-dive and merged summary, not from a candidate-specific adversarial review.

Implementation lens (round 2)

Supports
  • The desk profile is genuinely good — a trivial second oracle via exhaustive feature-grid closure (I2=2), a light calibration burden of two to three gates (I4=1), and no repair debt from round 1 (I6=2).
Against
  • No reusable repo exists, so I1 is a firm zero and, being double-weighted, dominates the total; first-match ordering partially couples the rules (I5=1) and a shadowed-rule audit is needed since a later rule may never fire (I3=1).

Research currency

Supports
  • Sits on live 2025–2026 research — RuleArena (ACL 2025) and the 2026 DeonticBench both frame rule-guided reasoning as an open problem, and LLMTabBench (2026) proposes a generator in exactly this shape.
Against
  • The core rule-list machinery is classical (CORELS is 2017), and the 2026 anchors are very recent with artifacts not independently corroborated, so the currency rests partly on unverified sources.

What would it take to build, and what could go wrong?

Implementation: steps & risks

Build the rule-list sampler

Programmatically sample a short ordered decision list over four to six categorical/numeric features with an explicit precedence chain of override rules — from scratch, since no reusable codebase was found.

Write the first-match executor and closure oracle

Execute the list top-to-bottom returning the first rule that fires; independently verify each label by exhaustively closing the finite feature grid (the trivial second oracle giving I2=2).

Add a shadowed-rule audit

Under first-match a later rule may be fully shadowed by an earlier one and never fire; check coverage so every rule owns a live case slice and the diagnostic ladder stays honest.

Relabel classes and balance strata

Defeat the pretrained triage prior with opaque label symbols, and balance the strata so 10–20-task internal evals resolve prompt differences.

Pin the ceiling prompt and floor gates

Author a state-all-rules ceiling prompt that provably reaches ~100% at temp 0; measure per-stratum floors; confirm no single fixed convention statement covers the pool (two to three pre-freeze gates).

Risks

  • No codebase means every hour is from-scratch — the double-weighted I1=0 is the single load-bearing reason this candidate ranks #17 despite a strong task design; there is no repo to inherit determinism, serialization, or an oracle from.
  • Shadowed rules silently break the ladder — if a later rule never fires under first-match, its stratum is dead, inflating apparent accuracy and mis-localizing failures; the shadowed-rule audit is mandatory.
  • First-match ordering couples the rules — precedence links rules, so changing one can perturb which cases the others govern, forcing more than a one-stratum regeneration (I5=1).
  • Over-simple encodings are one-shot solvable — without enough rule count, feature arity, and override depth, a competent prompt engineer closes the gap in one prompt.
