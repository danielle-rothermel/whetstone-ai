**Ranked #6 of 20 after the implementation-aware rescore — round-1's outright winner, still the best theoretical task in the pool.** In round 1 this was the single cleanest design: a straight sheet of top marks on the rubric and the only finalist both adversarial lenses left standing with nothing worse than minor, pinning-level caveats. ## What it proposes

### Task: relational/kinship QA over a nonce graph whose relation rules and answer words are both seeded secrets

A pinned generator builds a small graph of **nonce entities** connected by **nonce relation symbols**. Each relation symbol carries seeded, nonstandard **properties** fixed per generator seed: which relations are symmetric, which are transitive, an inverse mapping, a two-relation composition table, explicit-negation precedence, and closed-world handling. Answers use an **invented closed vocabulary** — a gender-neutral `nieph`, a `bond-` prefix for in-law-style links, a relation-distance cutoff. An instance is a few facts plus one query; the output is one or two words from the closed vocabulary or a 4-way MC letter (true / false / both / unknown), scored 0/1 exact match against a **graph-traversal oracle** that computes the answer under the seed's property assignments. Two independent sources of headroom stack: the model must learn the codebook *and* the traversal rules, and failures cleanly separate codebook misses (a standard English kin term slipped out) from traversal misses (the wrong relation rule was applied).

### Example instance

*One seed's conventions: `tovv` is transitive and non-symmetric; two `tovv` hops map to the codeword `nieph`; `brell` is symmetric; the closed-world default for `tovv` is `unknown`. (From recommendations.md; conventions are the seeded latent rules.)*

```
Facts:
- Marn tovv Belu.
- Belu tovv Sagre.
- NOT (Sagre brell Marn).

Q1: What is Sagre to Marn? Answer with one word from the task vocabulary.
Gold: nieph  (two tovv-hops). A naive prompt answers "grandchild" -> exact-match 0.

Q2: Does Belu tovv Marn? Answer A (true) / B (false) / C (both) / D (unknown).
Gold: D  (tovv is not symmetric under this seed; closed-world default for tovv is unknown).
A naive prompt guessing symmetry answers A -> 0.
```

## Perspectives

### Round-1 rubric fit
**Supports:** Straight top marks on all fourteen criteria — the hidden rulebook is arbitrary seeded information no zero-shot prompt can contain, so the low-floor/incremental-gap criterion is structural rather than tuned; roughly nine independent latent strata (six relation-property rules plus three vocabulary rules) give the finest-grained ladder in the pool.
**Against:** Even at a perfect scorecard, a best-effort "uninterpreted-symbols, closed-world literal logician" prompt could land mid-range if the hidden-rule strata are under-weighted — a floor that has to be pinned down by weighting, not assumed.

### Optimization-path skeptic
**Supports:** Not refuted — only minor caveats. The per-seed property assignments and invented codebook cannot be recalled or guessed and, with only a few facts per instance, are under-determined from any single instance, so "infer the rules yourself" prompts cannot shortcut the ladder.
**Against:** In-context rule induction via large demo sets is a real leakage path; the plan caps few-shot demo counts and adversarially tests an induce-the-conventions prompt precisely because that path exists.

### Measurement skeptic
**Supports:** Not refuted — only minor caveats. Failures are diagnostic: codebook misses are cleanly distinguishable from traversal misses, and within traversal the violated rule is identifiable from expected-vs-got.
**Against:** Some rules (the composition table, the distance cutoff) may be practically non-inducible from 0/1 feedback, which would stall every optimizer at the same sub-ceiling and falsify the success criterion for reasons unrelated to optimizer quality; multiple-valid-answer instances and an incoherent "both" label are latent generator bugs until rejection sampling removes them.

### Implementation lens (round 2)
**Supports:** Rides the same run-verified prontoqa base as c18 — the reusable-codebase axis is a full mark, skeptic-debt is verification-only (a full mark), and the c18 harness is roughly four-fifths of a future c21 harness, so the option stays cheap.
**Against:** The moving-part axis and the calibration axis both score zero — the most intricate generator in the set (graph/nonce sampler, relation-property profiles, unique-answer rejection sampler, MC/foil balance, stratified minibatches) plus four-plus pre-freeze gates. Both zeros are unique among eligible top-tier candidates and are what dropped it from first to sixth.

### Research currency
**Supports:** Sits squarely in the active in-context rule-learning / synthetic-ICL and rule-induction lines (cipher-ICL, GINC/RegBench, MIR-Bench, LINGOLY/Linguini) and is directly adjacent to PrOntoQA's fictional-symbol methodology; it could grow into a standalone optimizer-comparison benchmark.
**Against:** No flagship live leaderboard to anchor to — the tie-breaker's only noted gap — and its best-known genre anchors (CLUTRR, StepGame) are missing from its own reference set.

## Implementation: steps & risks

1. **Build the two pinned artifacts.** A graph sampler (nonce entities plus per-seed relation-property profiles) and a traversal oracle computing exact answers under those assignments; pin generator version, dataset manifest, and disjoint minibatch/subset/official-split identities. On the prontoqa base this maps to `--OOD --deduction-rule Composed --proofs-only --ordering random` (random ordering avoids the postorder crash) plus `--disjoint-concept-names` for the invented vocab.
2. **Cap hop depth and balance outcomes.** Tune hop depth down until the full-rule reference prompt reaches roughly 100%; balance the four MC outcomes and the closed-vocab answer distribution so constant-guess baselines pin low (25% MC floor).
3. **Tag strata and reject non-unique answers.** Tag every instance with its governing latent-rule stratum for failure attribution and stratified aggregation; rejection-sample any instance whose rule-closure admits more than one valid closed-vocab answer, and define "both" as arising only for relations whose seeded profile lacks negation-precedence.
4. **Run the pre-freeze gates.** Store the reference ~100% prompt, the naive baseline, and a literal-logician baseline weighted to score at or below half; run a per-rule discoverability check (feed a reflection LM failure traces per stratum, drop rules it cannot state); then pre-register a falsifiable success criterion.
5. **Normalize and score strictly.** Lowercase, strip whitespace, canonicalize hyphenated `bond-` terms; unparseable / off-vocabulary / empty outputs are decisive zeros. Add mean output UTF-8 bytes as a distinct second objective for two-objective coverage.

**Risks:**
- The generator is the pool's most intricate: graph/nonce sampler + relation-property profiles + unique-answer rejection sampler + MC/foil balance + stratified minibatches — the moving-part axis scores zero, so the effort is front-loaded and error-prone.
- Four-plus pre-freeze calibration gates (empirical ceiling, naive floor, literal-logician baseline, per-rule discoverability, adversarial induce-conventions) mean the calibration axis scores zero; if the discoverability check fails on the composition table or distance cutoff, those strata must be shrunk or dropped before freeze.
- On the prontoqa base, the Composed/relational path is exactly where the reachable `--ordering` postorder/preorder crash lives; the driver must force `--ordering random` (or apply the one-line `list(reversed(theory))` fix). The generator must also run from the repo root for `bad_patterns.txt`.

**Repo review & run verification:**
- Codebase: [asaparov/prontoqa](https://github.com/asaparov/prontoqa) (Apache-2.0) — deep-read notes at [../repos/asaparov-prontoqa.md](../repos/asaparov-prontoqa.md).
- Run verification: **partial (generator run-verified; c21's own adapted design never run-verified).** The prontoqa generator was run in a fresh venv — same-seed byte-identical, different-seed varies, three ground-truth hand-checks correct. That confirms the shared base, not c21's relational/invented-vocab strata, rejection sampler, or calibration gates, none of which have been built or run.
