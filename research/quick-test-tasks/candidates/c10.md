## What it proposes

### Task: advance a position in an invented calendar of coprime cycles by per-cycle modular counting

A seeded generator builds an invented calendar of two or three **coprime named cycles**
(Tzolk'in-style but randomized), plus a clock variant with positions labeled one through
*m*, **forward-only subtraction**, and **sticky-move** rules. An instance presents a
current tuple or clock position and an instruction to **advance N**; the model returns a
canonical tuple string or a single position, scored 0/1 by exact match against per-cycle
**modular arithmetic**. The latent rules that make the pool a ladder are: the moduli are
**independent** with no shared carry, labels are 1-indexed rather than 0-indexed, one
cycle carries a skip or **leap** rule, some moves are sticky or ignored, and the output
has a canonical format. Randomized cycle lengths and names per run make the pool
contamination-proof, and — the load-bearing property — a wrong-cycle answer isolates
which single modulus rule broke. Quirks are stratified so no single instruction sentence
covers the whole pool.

### Example instance

*Illustrative, constructed faithfully from the merged summary (the corpus ships no
run-verified seeded c10 instance — no repository implements the invented calendar). One
seed's conventions: three coprime cycles, all 1-indexed, one cycle with an every-other
advance.*

```
Calendar cycles (this seed): Vel = 1..13, Som = 1..7, Kir = 1..11.
Rule: Kir only advances on even-numbered steps (skip on odd steps).
All cycles are 1-indexed and carry independently (no shared carry).

Current: (Vel 12, Som 6, Kir 9).  Advance 5 days.

Per-cycle modular arithmetic (1-indexed):
  Vel: ((12-1 + 5) mod 13) + 1 = 4
  Som: ((6-1  + 5) mod 7 ) + 1 = 4
  Kir: advances on steps 2 and 4 of the 5 -> +2 -> ((9-1 + 2) mod 11)+1 = 10
Gold: (Vel 4, Som 4, Kir 10)

Naive miss: a model applying a shared decimal carry, 0-indexing, or
advancing Kir on all 5 steps returns the wrong cycle's value — and the
error names exactly which modulus rule it broke.
```

## Academic anchor & lineage

### Literature parents (inherited from c01)

c10's "Merged from (literature)" list is intentionally empty — the entry states its
modular-counterfactual evidence is covered under c01's sources, so its academic base is
entirely inherited.

- **[Reasoning or Reciting? Counterfactual Tasks](https://arxiv.org/abs/2307.02477)
  (Wu et al., NAACL 2024)** — the core inherited evidence: counterfactual two-digit
  arithmetic in nonstandard bases documents the low modular-notation floor. Verified
  directly that it contains no clock, calendar, or cyclic-temporal task, so the
  interlocking-cycle framing is a novel synthetic design, not a benchmark port.
- **[How well do LLMs perform in Arithmetic tasks?](https://arxiv.org/abs/2304.02015)
  (Yuan et al., 2023)** — non-decimal base arithmetic, documenting the
  convert-to-decimal-then-back failure mode relevant to per-cycle modular reduction
  errors.
- **[Teaching Algorithmic Reasoning via In-Context Learning](https://arxiv.org/abs/2211.09066)
  (Zhou et al., 2022)** — *(unverified: not re-fetched this pass)* c01-inherited:
  characterizes how prompt phrasing of an algorithm moves accuracy, supporting the
  incremental-gap claim for per-cycle advance and carry rules.
- **[NUMCoT](https://arxiv.org/abs/2406.02864) (Wang et al., 2024)** —
  *(unverified: not re-fetched this pass)* sibling c02 evidence for mixed-radix and
  time-unit conversion brittleness, adjacent to interlocking-cycle carry-free modular
  arithmetic.

## Perspectives

### Round-1 rubric fit

- **Supports:** Independent coprime moduli decompose difficulty into clean per-cycle
  rules and localize each failure to one modulus — strong on the criteria that matter
  most. Randomized cycle lengths and names per run are contamination-proof by
  construction.
- **Against:** Scored one point under the top rubric tier and, crucially, carried the
  low research-currency tie-breaker, which initially excluded it (folded back into the
  ranking by the 2026-07-21 amendment). Modular arithmetic is a pretraining-common
  skill, so without arbitrary secret conventions a competent one-shot prompt can
  approach the ceiling.

### Optimization-path skeptic

- **Supports:** Stratified interacting quirks (offset cycle, every-other advance, sticky
  moves) are meant to keep the modular-arithmetic prior from collapsing the gap in one
  shot.
- **Against:** No skeptic pass was ever run — c10 was initially excluded under the
  round-1 tie-breaker filter (folded into the ranking by the 2026-07-21 amendment), so
  this lens never adversarially examined it; the one-shot-collapse risk is asserted by
  the adaptation note, not stress-tested.

### Measurement skeptic

- **Supports:** Single-token canonical output is trivially 0/1 exact-match scorable, and
  the oracle is a five-line non-generator recompute hand-checkable in under a minute.
- **Against:** Never examined by the measurement skeptic while it was excluded under the
  tie-breaker filter (before the 2026-07-21 amendment folded it in); the
  landscape logs only a general cluster-wide caution to make conventions arbitrary and
  secret, not a c10-specific repair.

### Implementation lens (round 2)

- **Supports:** Ranked #5 of 20 on run-verified reusable infra: reasoning-gym's
  re-seedable ProceduralDataset infra, factory, and curriculum are reused as-is (repo
  run-verified under c24, carried over here); the coprime moduli mean adding a quirk
  touches only one modulus's stratum, so blast radius is contained (I5).
- **Against:** The invented interlocking-cycle generator and modular oracle are net-new
  authoring (~180-300 LOC) and default scoring is partial-credit and must be overridden —
  reuse-infra-write-task, not config plus glue, which caps I1.

### Research currency

- **Supports:** Test of Time (2024) shows constructed-cyclic temporal reasoning is a
  live, contamination-resistant direction the framing plugs into.
- **Against:** Carries the low tie-breaker: the design descends from counterfactual base
  arithmetic and its nearest published relatives sit just outside the 2025-2026 window,
  so it reads as less research-current than its siblings.

## Implementation: steps & risks

### Steps

1. **Vendor reasoning-gym, author a new generator.** Consume reasoning-gym as a
   dependency (no repo edits) and subclass `ProceduralDataset` in our tree; config
   carries sampled pairwise-coprime cycle lengths (gcd == 1), an offset/naming
   convention, and the latent-rule stratum id.
2. **Copy the seed idiom, write an independent modular oracle.** In `__getitem__` reuse
   the `rng = Random(seed + idx)` pattern from `calendar_arithmetic.py:129`; sample a
   start tuple and an advance N; compute the answer by fresh per-cycle modular arithmetic
   with 1-indexing and leap/skip adjustment — never stdlib `datetime`.
3. **Override scoring to strict 0/1 and add strata.** Override `score_answer` for
   single-token exact match (~5-10 LOC), bypassing the shipped partial-credit
   calendar/time scorers; optionally add a `BaseCurriculum` subclass exposing
   cycle-length ranges and convention families as strata.
4. **Gate before freeze.** Clear the two-to-three pre-freeze gates: verify the
   interacting quirks stop the modular-arithmetic prior from collapsing the gap one-shot,
   and stratify the arbitrary/secret conventions so no single prompt covers the pool.

### Risks

- The invented interlocking-cycle semantics are net-new logic — reasoning-gym ships only
  real-Gregorian tasks — so this is write-the-task, not config plus glue, and the reuse
  score is capped accordingly.
- Modular arithmetic is a strong pretrained prior; if the quirks are too few or too
  guessable a one-shot instruction closes the gap, collapsing the incremental ladder (the
  adaptation note's central caution, never stress-tested by a skeptic).
- The shipped `score_answer` on both calendar tasks awards graded/continuous credit and
  the base default gives substring credit — reusing any of them would silently break the
  strict 0/1 requirement.
- The reusable reasoning-gym infra is repo-run-verified (carried over from c24), but the
  candidate-specific interlocking-cycle task logic here is desk-estimated, not measured
  behavior of a built generator.

### Repo review & run verification

- **Codebase:** [open-thought/reasoning-gym](https://github.com/open-thought/reasoning-gym)
  (Apache-2.0) — deep-read notes at [../repos/reasoning-gym.md](../repos/reasoning-gym.md).
  Strongest architectural match: deterministic re-seedable ProceduralDataset generators,
  an exact-match oracle, curriculum/strata support, and register-and-go plumbing, with
  existing (real-Gregorian) calendar and time-interval tasks.
- **Run verification — repo run-verified (carried over from c24); task logic authored on
  top.** The reasoning-gym infrastructure was run-verified during round 2 under c24
  (same-seed byte-identical, 6/6 independent ground-truth checks; also verified
  deterministic per seed over five items, different seed varies, Gregorian ground truth
  hand-checked, 15/15 tests pass), and that repo-level verification carries over here.
  The invented interlocking-cycle / coprime-modulus task c10 demands does not exist
  in-repo and its candidate-specific logic is authored on top of the reused infra, so
  I1 is held at 2 for this reason.
