# c23 · Hidden-Rule String-Transform Induction Ladder

**Status:** RANKED #11 of 20 (round-2 implementation rescore)

**You know this as:** compositional rule induction from I/O demonstrations — the InductionBench / PCFG-SET / MiniSCAN family, where a model must infer K latent string-transform rules from few-shot examples and apply them to a held-out query.

---

## Verdict

**Ranked #11 of 20 after the implementation-aware rescore, up from the 26-tier in round 1.** The task design is one of the cleanest in the pool: an independent oracle applies a latent rule set, the rules decompose into inspectable strata, few-shot demos are load-bearing by construction, and randomized rule subsets over invented vocabularies make it contamination-proof. What holds it back is the codebase, not the concept. The one candidate repo — InductionBench — has a genuinely independent, self-consistent oracle worth re-implementing, but it does not import or run as shipped: three import-time crashes were verified by execution, and its "seeded" generator is not actually reproducible unless `PYTHONHASHSEED` is pinned. The failed-run cap therefore holds the reuse score at its floor, and that double-weighted penalty is the whole reason a design this clean lands mid-pack rather than on the podium.

**Scores** — R1 rubric/TB: 26/2 · I1: 1 · I2: 2 · I3: 1 · I4: 1 · I5: 2 · I6: 2 · Impl total: 10/16.

## What it proposes

### Task: induce K independent latent transform rules from demos, apply to one query

The designer defines K independent transformation rules — casing keyed to token length, positional swaps, subregular ISL/OSL string edits, per-primitive command→action mappings, or 1D-ARC-style short-array transforms. Each seeded instance applies a **rule subset** to an input. The model sees a handful of few-shot **input→output demonstrations** plus one **query**, and must output the transformed short string — or pick from a 4-way multiple choice among candidates each wrong in exactly one rule. Scoring is **exact match** against the reference transformer, 0/1 with no partial credit. Prompt quality equals how many rules the prompt states or induces, giving a smooth incremental **ladder** with a known ceiling (state all rules) and floor (naive prompt). Because demos are the only channel for the latent rules, the few-shot axis is alive by construction; randomized rule subsets over invented vocabularies keep it contamination-proof.

### Example instance (illustrative)

*Seeded conditions: two latent ISL/OSL-style rules over a small invented vocabulary, applied together to each demo; the query exercises both.*

```
Demonstrations (rules: (1) double any token ending in "z"; (2) uppercase the first token):
  vez fam qor   ->  VEZ vez fam qor
  qor luz tam   ->  QOR tam luz luz
  fam vez luz   ->  FAM vez luz luz

Query:
  luz qor vez   ->  ?

Gold answer:
  LUZ luz qor vez        (rule 2 uppercases "luz"; rule 1 doubles it because it ends in "z")

Naive-prompt miss:
  "Repeat the transformation shown"  ->  LUZ qor vez
  (finds rule 2 from the visible capitals, misses the length/suffix-keyed doubling — earns 0 on exact match)
```

## Academic anchor & lineage

### Literature parents

- **InductionBench (Hua et al., ACL 2025; arXiv:2502.15823)** <https://arxiv.org/abs/2502.15823> — the central design driver: subregular ISL/OSL string-function induction from I/O demos, with the reachability caveat that frontier models fail even the simplest complexity class.
- **PCFG SET / Compositionality Decomposed (Hupkes et al., JAIR 2020; arXiv:1908.08351)** <https://arxiv.org/abs/1908.08351> — string-edit transduction with five named generalization axes; the direct template for c23's per-operator independent latent rules.
- **SCAN (Lake & Baroni, ICML 2018; arXiv:1711.00350)** <https://arxiv.org/abs/1711.00350> — foundational compositional command→action grammar induction; c23's MiniSCAN single-step stratum descends from it.
- **COGS (Kim & Linzen, EMNLP 2020; arXiv:2010.05465)** <https://arxiv.org/abs/2010.05465> — compositional-generalization semantic-parsing benchmark; source of c23's generalization-split framing (SLOG, EMNLP 2023, arXiv:2310.15040, extends it).
- **Instruction Induction (Honovich et al., ACL 2023; arXiv:2205.10782)** <https://arxiv.org/abs/2205.10782> — few-examples→task-description induction suite (APE, Zhou et al., arXiv:2211.01910, reuses its tasks); c23's instruction-induction lineage.
- **List Functions benchmark (Alet et al., ICML 2021)** <http://proceedings.mlr.press/v139/alet21a.html> — few-shot list→list program induction (BIG-bench `list_functions`); c23's numeric-bounded induction stratum.
- **Re-ARC (Hodel, 2024; arXiv:2404.07353)** <https://arxiv.org/abs/2404.07353> — procedural ARC generator with a difficulty dial; paired with 1D-ARC for c23's linearized short-array stratum.

### Brainstorm parents

- **Compositional String Decode (CSD)** — the brainstorm seed; the origin note in [candidates-merged.md](../candidates-merged.md) records c23 as "origin: both" (brainstorm + literature).
- Merged entry: [c23 in candidates-merged.md](../candidates-merged.md) folds the CSD brainstorm item together with the InductionBench / PCFG-SET / Instruction-Induction / List-Functions literature cluster into one ladder.

### Dead branches: dropped near-relatives

- **D4 · CRUXEval-style code-output prediction** — the published set is heavily contaminated and regeneration needs a code DSL with unconstrained return types that fight exact-match bounding; its rule-induction flavor is explicitly folded into c23.
- **D5 · Grounded grid compositional (gSCAN / ReaSCAN)** — grid rendering plus action-sequence outputs are the most expensive to bound, and the reviewer judged it "not worth the adaptation cost"; its compositional-splits flavor is folded into c23 (the grid flavor went to c19).
- **HERO'S JOURNEY / WILT / ARISE** — named in the merge as generators only, not adopted as tasks: their multi-step episodic form violates the single-call constraint, and the procedural-induction family risks an unreachable ceiling.

## Perspectives

### Round-1 rubric fit
- **Supports:** Difficulty decomposes into K independent, nameable latent rules that a prompt either states or misses, so the naive floor sits well below the write-out-all-rules ceiling and the gap closes in steps.
- **Supports:** Fully generator-native with exact ground truth and invented vocabularies, so it is contamination-proof and produces hundreds of instances cheaply.
- **Against:** InductionBench's own finding that frontier models fail the simplest complexity class means the ceiling may be unreachable unless rule complexity is deliberately curated — a real threat to the "close the gap" story.

### Optimization-path skeptic
- **Supports:** Demos are the only channel for the latent rules, so the few-shot / MIPROv2 axis is alive by construction rather than bolted on.
- **Against:** A single-rule or overly transparent stratum is one-shot guessable — "it's a substitution/casing rule" can close the gap in one sentence — so the ladder only holds if rules are genuinely composed.

### Measurement skeptic
- **Supports:** The oracle is genuinely independent of the sampler — rules are sampled by one function and outputs computed by a separate deterministic transducer — and verified self-consistent across configs, so ground truth is not tautological (drives I2=2).
- **Against:** Exact-match on short transformed strings is legitimate but per-instance binary, so a small eval needs enough query items to resolve a 10-point prompt change above noise.

### Implementation lens (round 2)
- **Supports:** The generation-plus-oracle core is small (~430 LOC on-path) and, once patched, reusable; strata are independent so a change regenerates one stratum with a local diff (I5=2) and the round-1 skeptic verdict left only verification-level debt (I6=2).
- **Against:** The repo does not import or run as shipped — a missing `config.py`, a `sys.path.add` crash, and an import of a nonexistent function are three verified import-time failures — and its generator is nondeterministic unless `PYTHONHASHSEED` is pinned, which caps reuse at the floor (I1=1) and forces vendoring plus a seed refactor.

### Research currency
- **Supports:** In-context rule induction is unusually active in 2025-2026 (InductionBench, MIR-Bench, FalsifyBench, HERO'S JOURNEY), so a quick-test built on this generator has a natural path to becoming its own research artifact.
- **Against:** The freshest members of this family are also the hardest, so borrowing their difficulty framing risks importing their unreachable-ceiling problem.

The skeptic column above is reconstructed from the merged summary and the round-2 desk scoring rather than from a standalone round-1 adversarial review.*

## Implementation: steps & risks

1. **Vendor the two on-path files.** Copy `synthetic_data_generation.py` (~330 lines: generation + oracle) and the three needed functions from `utils.py` (~40 lines) into our repo under Apache-2.0 with the NOTICE preserved. Do **not** vendor `inference.py`, `model.py`, or `standard_run.py`.
2. **Fix the three import-time blockers.** Add a real `config.py` (or refactor `config.vocab` into an explicit argument), delete the `translate_fewshot_input_output_pairs` import at `synthetic_data_generation.py:8`, and drop the entire few-shot path — roughly a five-line set of edits.
3. **Thread a real seed.** Add a `seed` parameter to `generate_rules` / `generate_data`, replace the module-global `random.seed(0)`, and either pin `PYTHONHASHSEED=0` in the launcher or replace every `list(set(...))` over strings with `sorted(...)` at the ~six call sites (lines 53, 61, 107, 132, 208, 244). The `sorted(...)` route is preferred because it kills hash-order nondeterminism at the source.
4. **Write the glue outside the repo (~60-100 lines).** An instance loop calling `generate_rules` + `generate_data` per stratum (varying `--type` / `--k` / `--vocab_size` / `--number_of_rules`), a single-LLM-call driver, a 0/1 exact-match scorer comparing the model's applied output against the oracle, a latent-rule strata config, and optional nonce-vocab remapping.
5. **Calibrate reachability before freezing.** Verify a reference prompt hits ~100% on the easiest strata so the incremental gap is real headroom and not "nobody succeeds," curating rule complexity against InductionBench's simplest-class-fails warning.

### Risks

- **Ceiling unreachable:** InductionBench reports even o1/o3 fail the simplest subregular class, so without deliberate complexity curation the write-out-all-rules ceiling may sit below 100% and collapse the incremental-gap story.
- **Nondeterminism leaks past the seed:** the generator's `list(set(...))` over strings makes output depend on `PYTHONHASHSEED`; if the seed refactor is incomplete the "seeded" pool is not reproducible, breaking same-seed re-runs.
- **One-shot guessable strata:** a transparent single-rule stratum can be solved by one sentence of prompt, so the ladder must lean on composed, wrong-in-exactly-one-rule distractors to preserve partial-credit structure.

### Repo review & run verification

- **Codebase:** [Wenyueh/inductive_reasoning_benchmark (InductionBench)](https://github.com/Wenyueh/inductive_reasoning_benchmark) (Apache-2.0) — deep-read notes at [repos/inductionbench-wenyueh-inductive-reasoning-benchma.md](../repos/inductionbench-wenyueh-inductive-reasoning-benchma.md).
- **Run verification:** **failed** — the active path does not import as shipped (three verified import-time crashes: missing `config.py`, `sys.path.add` `AttributeError` at `inference.py:4`, and an `ImportError` for a nonexistent `translate_fewshot_input_output_pairs`), and two same-seed runs produced different instance hashes until `PYTHONHASHSEED=0` was pinned. The oracle was spot-checked as self-consistent (re-applying each latent rule reproduced stored outputs), but the failed run verification caps I1 at 1.
