**Status: BACKUP** — officially ranked #2 of 20 in the round-2 rescore.

*You know this as:* RuleTaker / ProofWriter / PrOntoQA-style depth-binned synthetic
deduction — sampled facts plus if-then rules over fictional predicates, queried for a

[asaparov/prontoqa](https://github.com/asaparov/prontoqa) generator was run-verified

## What it proposes

### The task: classify a multi-hop entailment at a dialed-in proof depth

A seeded generator regenerates RuleTaker/FLD/PrOntoQA-style **theories** — sampled
**facts** plus **if-then rules** over **fictional predicates** ("every wumpus is a
yumpus") — with **controlled proof depth** (D0-D5), **distractor rules**, and closed-
versus open-world semantics. Each instance is a theory plus a **query**; the model
emits **True / False / Unknown**, scored by exact match. Ground truth comes from the
generator's **proof engine**, which additionally names the **failed inference step** for
reflection diagnostics. The **latent rules** are chaining depth, negation-as-failure
versus explicit negation, open-world Unknown, exceptions, and quantifiers; the depth
bins give both the incremental ladder and a known ceiling. A **unique-solution
constraint-puzzle** stratum (a seating arrangement queried for one fact, name or YES/NO
output) covers the CSP flavor. Fresh symbols per run make it contamination-proof.

### Example instance (illustrative, hand-traced from a run-verified prontoqa output)

Seeded conditions: `--ontology fictional`, two-hop depth, seed 12345. The nonce
predicates and the negation flag are drawn from the seed, so the surface text carries no
real-world prior.

```
Facts & rules:
  Wren is an impus.
  Every impus is a vumpus.
  Vumpuses are not loud.
Query:   Wren is not loud.
Gold:    True     (impus -> vumpus -> not loud; a two-hop chain)

Naive-prompt miss: a model that answers from surface plausibility, or that treats
"not loud" under the wrong world assumption, returns False/Unknown; only following
the two stated rules in order yields True.
```

## Academic anchor & lineage

### Literature parents

- **[RuleTaker / Transformers as Soft Reasoners over Language](https://arxiv.org/abs/2002.05867)** (Clark, Tafjord & Richardson, IJCAI 2020) — the core template: synthetic facts plus if-then rules, depth-binned D0-D5 theories, True/False under closed-world assumption with negation-as-failure.
- **[ProofWriter](https://aclanthology.org/2021.findings-acl.317/)** (Tafjord, Dalvi & Clark, Findings of ACL-IJCNLP 2021) — the RuleTaker successor adding open-world semantics with an explicit Unknown, the source of the T/F/Unknown three-way label.
- **[PrOntoQA / Language Models Are Greedy Reasoners](https://arxiv.org/abs/2210.01240)** (Saparov & He, ICLR 2023) — the fictional-ontology generator ("every wumpus is a yumpus") behind the reused codebase; contamination-resistant symbols and proof-step failure diagnosis.
- **[PrOntoQA-OOD](https://arxiv.org/abs/2305.15269)** (Saparov et al., NeurIPS 2023) — the OOD depth/compositionality extension supplying the incremental-ladder and headroom evidence.
- **[FLD / Learning Deductive Reasoning from Synthetic Corpus](https://arxiv.org/abs/2308.07336)** (Morishita et al., ICML 2023) and its follow-up **[FLD*](https://arxiv.org/abs/2411.12498)** (Morishita et al., NeurIPS 2024) — a principled complete-rule-set generator with proved/disproved/unknown labels under controllable depth and distractors.
- **[LogicNLI](https://aclanthology.org/2021.emnlp-main.303/)** (Tian et al., EMNLP 2021) — controlled FOL-NLI with a small AST inventory and depth up to five; an abstract/templated regeneration model.
- **[SATBench](https://arxiv.org/abs/2505.14615)** (Wei et al., EMNLP 2025) — the recent hard tail: a SAT/CNF-to-puzzle generator with a solver oracle and documented large headroom (o4-mini near random on hard UNSAT).
- **RuleBERT** (Saeed, Ahmadi, Nakov & Papotti, EMNLP 2021, https://aclanthology.org/2021.emnlp-main.110/) — the soft-rule stratum; and **FOLIO** (Han et al., EMNLP 2024, https://aclanthology.org/2024.emnlp-main.1229/; preprinted arXiv:2209.00840, 2022) held as a **style reference only** (human-written, ~1.4k items, not synthetically scalable).

### Brainstorm parents

- Constraint-Satisfaction Seating Puzzle (unique-solution stratum) — the brainstorm seed folded in as the CSP-flavor stratum of the merged candidate.
- The [candidates-merged entry](../candidates-merged.md) records c18 as origin "both" (brainstorm and literature), merging the seating puzzle with the RuleTaker/ProofWriter/FLD/PrOntoQA literature cluster.

### Dead branches: dropped near-relatives

- **FOLIO as a scalable source** — downgraded to *style reference only*: human-written, roughly 1.4k items, not synthetically regenerable, so it cannot anchor a seeded infinite pool.
- **Reuse of published RuleTaker/ProofWriter/PrOntoQA sets verbatim** — refused because the released theories are contaminated; the adaptation must regenerate with fresh symbols rather than reuse.
- **Full proof-chain generation as output** — dropped from the design; the adaptation keeps the label only, to bound output tokens and preserve exact-match scoring.
- (Not this candidate: the `LLMStructBench` unverifiable-provenance flag belongs to c12's extraction cluster, not c18 — no laundering needed here.)

## Perspectives

### Round-1 rubric fit
- **Supports:** scored 27 with a tie-break of 2; landscape names it "the cleanest incremental ladder" via proof-depth bins, and the proof engine names the failed step for reflection.
- **Against:** it sat just below the 28-tier finalist cut, held back by the heavier reasoning load per instance.

### Optimization-path skeptic
- **Supports:** fresh symbols per run and depth binning make the decisive knowledge (chaining depth, negation-as-failure, open-world Unknown) discoverable from demos rather than one-shot guessable, and the depth ceiling is designer-known.
- **Against:** this candidate never received a dedicated round-1 skeptic pass — the adversarial optimization-path and measurement lenses were run on the seven 28-tier finalists, not on 27-tier tasks, so no skeptic verdict below the finalist tier examined it.

### Measurement skeptic
- **Supports:** the label is a single T/F/Unknown token under exact match, and the run verified 3/3 hand-traced labels correct with valid proof chains derivable purely from in-instance rules — no hidden knowledge.
- **Against:** the emitted label is *definitional* (it comes from a 50%-probability negation flag), not an independent theorem-prover verdict, so a soundness bug in generation would not be caught by the label alone; and CWA-versus-Unknown semantics can make the Unknown/False boundary ambiguous without a precise reference prompt.

### Implementation lens (round 2)
- **Supports:** the codebase is run-verified deterministic (byte-identical at seed 12345, varying at 999) and the adaptation is ~100 lines of external glue with no required repo edits (I1=3).
- **Against:** the design breadth costs it — a second constraint-puzzle stratum plus CWA/Unknown pinning keep the harness-shape and gate scores low (I3=1, I4=1, I5=1), so the total lands mid-pack rather than at the top.

### Research currency
- **Supports:** the deductive-reasoning line is alive in 2025-2026 (JustLogic 2025, SATBench EMNLP 2025, FLD* NeurIPS 2024), and prontoqa is actively cited in current reasoning-eval papers.
- **Against:** the anchor generators (RuleTaker 2020, ProofWriter/LogicNLI 2021, PrOntoQA 2023) are several years old, and the task shape is well-trodden rather than novel.

## Implementation: steps & risks

1. **Subprocess generation per stratum** — call `run_experiment.py --model-name json` per depth stratum, looping `--min-hops/--max-hops` for the D0-D5 ladder, `--ontology fictional` for nonce vocabulary, `--distractors {none,relevant,irrelevant}` and deduction rule as difficulty knobs. Run from the repo root so the relative `bad_patterns.txt` import resolves. (~40 lines of driver plus an arg table.)
2. **Parse the emitted JSON into the instance schema** — read `example{i}.test_example.{question,query,answer}`, treating `answer` as the 0/1 exact-match gold and dropping the optional `chain_of_thought`. (~30 lines.)
3. **Single scored LLM call** — concatenate `question` plus `query`, ask for True/False/Unknown, exact-match against `answer`. (~20 lines, oracle-independent of the model under test.)
4. **Add the independent oracle and the CSP stratum** — write a ~30-50-line forward-chaining fixpoint check so the label is re-derived rather than trusted as the definitional negation flag, and add the unique-solution seating stratum for the CSP flavor.

**Risks**

- The T/F/Unknown label is definitional (a negation flag), not an independent prover verdict, so a generation bug is invisible in the label alone — mitigate with the ~30-50-line forward-chaining fixpoint check above.
- CWA-versus-Unknown semantics needs a precise reference prompt (2-3 pre-freeze gates) or the Unknown/False boundary is ambiguous.
- `--deduction-rule Composed` with postorder ordering crashes (`TypeError` on `reversed(theory)`, verified, 0-byte output) — use `--ordering random`; and small concept pools with high hop counts loop on retry-by-rejection, so size the vocab to the requested depth.

**Repo review & run verification**

- **Codebase:** [asaparov/prontoqa](https://github.com/asaparov/prontoqa) (Apache-2.0, maintained, actively cited) — deep-read notes at [../repos/asaparov-prontoqa.md](../repos/asaparov-prontoqa.md).
- **Run verification: verified.** `--seed` threads into both `random` and `np.random`; two runs at seed 12345 were byte-identical (md5 `c5e93ed...`), seed 999 differed, and 3/3 hand-traced labels plus proof chains were correct and derivable purely from in-instance rules. Only new caveat: must run from repo root for `bad_patterns.txt`.
