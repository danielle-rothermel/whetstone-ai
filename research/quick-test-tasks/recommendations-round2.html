Methodology: codebase reuse is the primary sort key

The I1–I6 rubric (higher = easier/safer)

  • I1 (0–3, double-weighted) — reusable existing codebase. 3 = maintained; permissive or paper-attached license; used in 2025–2026 work; diff is config + scoring glue; active path well-written and verified by a direct deep code read — never README/stars alone. 2 = reusable, one property missing. 1 = major surgery / partial frozen generator. 0 = no usable codebase.
  • I2 (0–2) — oracle independence. 2 = truth checkable via a second independent path and hand-verifiable in under a minute.
  • I3 (0–2) — moving parts of the adapted design, incl. round-1 mandated foil banks, rejection samplers, discoverability checks.
  • I4 (0–2) — calibration burden: pre-freeze empirical gates (ceilings, floors, audits, red-team runs). 2 = 0–1; 1 = 2–3; 0 = 4+.
  • I5 (0–2) — change-tolerance: 2 = a rule change regenerates one stratum with a local manifest diff.
  • I6 (0–2) — redesign debt from round-1 verdicts: 2 = verification-only; 1 = bounded; 0 = structural.
  • TOTAL = 2·I1 + I2 + I3 + I4 + I5 + I6, max 16.

Weighting, gating, and how scores were produced

  • Why I1 is doubled: reusing a clean benchmark codebase already in active use — which makes fast publishing much likelier — is roughly 2× more important than a task being theoretically somewhat nicer. Implementation total is the primary sort; round-1 score/TB only break ties.
  • Verdict health is a gate, not a score: candidates round 1 refuted without a concrete rescue (c01, c05, c07, c15) cannot be recommended at any implementation score. Bounded-redesign candidates stay eligible; I6 prices their debt.
  • Pool filter — applied initially, dropped by amendment: round 2 initially ranked only the 21 candidates with round-1 tie-breaker 2; the 2026-07-21 amendment drops that exclusion by user decision, folding c02/c04/c10 in on equal terms, their cells produced by the same pipeline.
  • Production: desk scoring of I2–I6 from round-1 docs with cited evidence → web discovery per candidate → deep active-path code reads by dedicated agents (seed plumbing traced, tests run, red flags at file:line) → run verification of finalists (same-seed byte-identical, different-seed varies, independent ground-truth spot checks) → I1 finalized: failed run caps at 1, confirmed run keeps, no repo = 0.
  • I1 adjustments: c18/c19/c21 confirmed at 3; c01/c11/c17/c24 confirmed at 2; c07/c23 held at 1 (InductionBench: three verified import-time crashes + PYTHONHASHSEED nondeterminism); c12/c13/c14/c16 = 0 (no reusable repo found); unverified provisionals kept as read. Amendment fold-ins: c04 = 3 (reasoning-gym's ab.py is a genuine term-rewriting-to-normal-form generator; repo-level run verification carries over from c24, with the seeded rule-table generalization authored on top); c10 = 2 (same run-verified base, but the invented interlocking-cycle task exists nowhere in-repo and must be authored on top); c02 = 1 (NUMCoT: unseeded dead-code generator, provably buggy solve_hard oracle — never run-verified, below the I1≥2 bar).

1 · c19 — Grid-World State Prediction (14/16)

adopt Deterministic spatial simulation · round-1 27/2 · the only run-verified I1=3 codebase whose skeptic debt is verification-only.

Reusable codebase: Minigrid, zero repo edits

Farama-Foundation/Minigrid — Apache-2.0, v3.1.0, actively maintained. Deep read + run verification: repos/farama-foundation-minigrid.md. Seed threads reset(seed) → self.np_random through all generation; no module-global seeding. Run-verified: seed 42 byte-identical SHA-256 across runs on five envs, seed 43 differs, 3/3 ground-truth spot checks via an independent object-model walk.

Adaptation diff: ~120–200 lines of glue in our package — seeded instance gen (reset(seed) + pprint_grid/grid.encode), independent derived-fact oracle, strata over env id/size, optional nonce symbol remap, single LLM call + strict 0/1 scorer.

Remaining risks are ours, not the repo's

  • The invented-command / wrap-vs-clamp convention strata that defeat one-shot guessing are net-new simulation-layer logic; Minigrid supplies generation, serialization, determinism — not nonstandard dynamics.
  • Use stochastic-layout envs (Fetch, Crossing, FourRooms, Empty-Random); fixed Empty-8x8 does not vary by seed.
  • Avoid off-path unseeded spots (wrappers.py:800, WFC subpackage).
  • 2–3 pre-freeze gates: measured ceiling, per-stratum floors, single-fixed-convention red-team.
  • License-metadata mismatch (LICENSE=Apache-2.0, pyproject says MIT) — cite the LICENSE file.
