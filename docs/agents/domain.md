# Domain Docs

Configured layout: **single-context**.

This repository uses a single domain context.

## Before exploring

- Read `CONTEXT.md` at the repository root when it exists.
- Read relevant decisions under `docs/adr/` when that directory exists.
- If either is absent, proceed silently; domain-modeling creates it lazily when a real term or decision is resolved.

Keep domain docs canonical at these paths. Versioned plans link to them rather than copying them into planning packets. Use glossary vocabulary consistently and flag conflicts with existing ADRs explicitly.
