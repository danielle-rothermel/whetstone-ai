# Issue tracker: Linear

Plans, priorities, and backlog for this repo live in Linear under the DevInfra team and Code Compression Paper project.

## Conventions

- Use Linear issues as the live decision store.
- Repository docs under `docs/` are reference material, not active task lists.
- External pull requests are not a triage request surface.
- Wayfinder maps and child tickets should be represented as linked Linear issues with explicit blocking relationships.

## Skill operations

- When a skill says to publish to the issue tracker, create a Linear issue in the configured team and project.
- When a skill says to fetch a ticket, read the referenced Linear issue and its comments.
- Claim work before starting and record durable conclusions on the issue.
