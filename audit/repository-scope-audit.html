Whetstone is currently a HumanEval platform application, not a standalone package

whetstone-ai · repository scope audit · 2026-07-15
The repository fails every requested boundary except one narrow negative finding: there is no tracked Python plotting implementation. DuckDB, MotherDuck, Unitbench, visualization dependencies, integration tests, platform operations, HumanEval treatment logic, fixed data projections, and extensive recovery machinery are present. Generated artifacts were removed on main after the original audit. The remaining work is an architectural contraction, not a small cleanup.

A safe contraction removes whole vertical slices first

Delete the Unitbench delivery slice

Remove the release-parity workflow, production fixture, publication CLI subcommands, associated tests, secrets, and integrity-only dependencies.

Delete DuckDB, MotherDuck, and data projection as one slice

Remove the fixed publication module, remote/local bundle behavior, COPRO bundle reader path, MotherDuck connection policy and tests, then regenerate the lock and confirm rg -i 'duckdb|motherduck' is empty.

Delete visualization and experiment residue

Remove Marimo and Matplotlib dependencies, all tracked artifacts, and committed HumanEval experiment/model/split configurations.

Remove the platform application

Delete DBOS workflows, targets, worker, enqueue runtime, operations, acceptance, cutover, live sweep, persistence, PostgreSQL schema/migrations, and the explicit integration suite.

Define one standalone public contract

Choose the smallest reusable behavior—likely graph/LM request execution and typed outcomes—export it from whetstone, document it, and make benchmark, provider, persistence, and orchestration adapters explicit and optional.

Simplify records and failures around that contract

Remove HumanEval metrics and defaults, redundant identities, storage-size policy, dynamic failure exports, third-party exception heuristics, and silent fallbacks. Validate only at real input boundaries.

Rebuild tests from public behavior

Retain deterministic request/response, parsing, sanitization, and explicit error tests. Delete source inspection, workflow text, import-order, crash-boundary, database, runtime, and downstream scenario tests.

Do not begin by deleting individual checks. Most ceremony belongs to removable vertical slices. Removing the slices first avoids spending time deciding whether each journal checksum, retry state, or cleanup proof is locally justified.
